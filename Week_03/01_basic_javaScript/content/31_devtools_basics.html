ความรู้เบื้องต้นเกี่ยวกับ Browser DevTools

Browser Developer Tools (DevTools) คือชุดเครื่องมือสำหรับนักพัฒนาที่ถูกติดตั้งมาพร้อมกับเว็บ Browser เป็นเครื่องมือที่ทรงพลังอย่างมากในการตรวจสอบโครงสร้างหน้าเว็บ (Inspect), แก้ไขข้อผิดพลาด (Debug JavaScript), ตรวจสอบเครือข่าย (Network) และวิเคราะห์ประสิทธิภาพการทำงานของเว็บไซต์ แม้ว่าบทเรียนนี้จะอ้างอิงจาก Google Chrome เป็นหลัก แต่แนวคิดและวิธีการใช้งานสามารถนำไปปรับใช้กับ Browser อื่น ๆ (เช่น Safari, Firefox, Edge) ได้เช่นกัน

1. วิธีการเปิดใช้งาน DevTools

  1. ใช้คีย์ลัด (Keyboard Shortcuts): บน Windows/Linux กดปุ่ม F12 หรือ Ctrl + Shift + I และบน Mac กดปุ่ม Cmd + Option + I
  2. คลิกขวา ที่องค์ประกอบ HTML ใด ๆ บนหน้าจอ แล้วเลือกเมนู "Inspect" (ตรวจสอบ)
  3. เปิดผ่านเมนู Browser: คลิกที่ปุ่มจุดสามจุดมุมขวาบนของ Chrome -> เลือก More Tools (เครื่องมือเพิ่มเติม) -> Developer Tools (เครื่องมือสำหรับนักพัฒนาซอฟต์แวร์)

ภาพรวมแท็บเครื่องมือหลัก ๆ

  1. Elements: ใช้สำหรับตรวจสอบและแก้ไขโครงสร้าง HTML และสไตล์ CSS แบบเรียลไทม์
  2. Console: ใช้สำหรับรันโค้ด JavaScript ทดสอบคำสั่ง และดูข้อความ Log หรือข้อผิดพลาด (Errors)
  3. Network: ใช้สำหรับตรวจสอบการรับส่งข้อมูลผ่านเครือข่าย (เช่น การดึงข้อมูลจาก API)
  4. Application: ใช้สำหรับตรวจสอบข้อมูลที่ Browser บันทึกไว้ (เช่น Local Storage, Session Storage, Cookies)
  5. Sources: ใช้สำหรับดูไฟล์ Source Code ทั้งหมด และทำระบบ Debug โค้ดทีละบรรทัด (Breakpoints)
  6. Performance: ใช้สำหรับวิเคราะห์ความเร็วและประสิทธิภาพการแสดงผลของหน้าเว็บ

2. แท็บ Elements (การจัดการโครงสร้างและสไตล์หน้าเว็บ)

ฟีเจอร์เด่นที่ใช้งานบ่อย:

  1. สำรวจโครงสร้างหน้าเว็บ: สามารถไล่ดู Tag HTML ต่าง ๆ และเมื่อนำเมาส์ไปชี้ ระบบจะไฮไลต์พื้นที่ขององค์ประกอบนั้นบนหน้าจอจริงให้เห็น
  2. แก้ไข HTML แบบเรียลไทม์ (Live Edit): สามารถดับเบิลคลิกเพื่อแก้ข้อความ หรือคลิกขวาเพื่อลบ คัดลอก หรือซ่อน Element เพื่อทดสอบหน้าเว็บได้ทันที
  3. ปรับแต่ง CSS (Styles Panel): แสดงสไตล์ CSS ทั้งหมดที่ส่งผลต่อ Element นั้น สามารถเปิด/ปิดสไตล์ แก้ไขค่าสีผ่าน Color Picker และตรวจสอบขนาดผ่าน Box Model ได้
  4. ตรวจสอบ Event Listener: ดูรายการเหตุการณ์ (Events) ทั้งหมดที่ถูกผูกไว้กับ Element นั้น
  5. จำลองสถานะ (Force State): สามารถจำลองสถานะพิเศษ เช่น :hover, :active, หรือ :focus เพื่อดูผลลัพธ์ของ CSS ได้สะดวก

ตัวอย่างการตรวจสอบและแก้ไขสไตล์หน้าเว็บ

หากมีองค์ประกอบบางอย่างไม่แสดงผลบนหน้าจอตามที่ควรจะเป็น เราสามารถใช้เครื่องมือ Inspect เพื่อตรวจสอบว่ามีสไตล์อย่าง display: none หรือ visibility: hidden บังอยู่หรือไม่ หรือตรวจสอบระยะห่าง Margin/Padding ที่อาจจะคลาดเคลื่อน จากนั้นลองแก้ไขค่า CSS ในแผง Styles ได้ทันทีเพื่อดูผลลัพธ์

การค้นหาสไตล์ CSS ที่ขัดแย้งกัน

ในแผง Styles สไตล์ CSS ใดที่มีเส้นขีดทับ (Strikethrough) หมายความว่าสไตล์นั้นถูกสไตล์ตัวอื่นที่มีความสำคัญสูงกว่าเขียนทับ (Override) ไปแล้ว โดย DevTools จะแสดงสไตล์ตัวที่ชนะอยู่ด้านบน พร้อมระบุชื่อไฟล์และเลขบรรทัดที่เขียนสไตล์นั้นไว้อย่างชัดเจน

3. แท็บ Console (การทดสอบโค้ดและดูข้อผิดพลาด)

แท็บ Console เป็นพื้นที่ยอดนิยมสำหรับนักพัฒนาซอฟต์แวร์ ใช้สำหรับตรวจสอบข้อผิดพลาด (Errors) ที่เกิดขึ้น และยังทำหน้าที่เป็นสนามทดลองเขียนโค้ด JavaScript เพื่อสั่งงานหน้าเว็บได้โดยตรง:

document.title = "New Title"
document.body.style.backgroundColor = "lightblue"
const button = document.querySelector("button")
button.style.fontSize = "20px"
function add(a, b) { return a + b }
add(5, 3) // 8

💡 เกร็ดความรู้: หาก Browser ขึ้นคำเตือนเพื่อความปลอดภัยเมื่อคุณพยายามวางโค้ดลงใน Console ให้พิมพ์คำว่า allow pasting ลงไปแล้วกด Enter ก่อนเพื่ออนุญาตให้วางโค้ดได้

4. แท็บ Network (การตรวจสอบการรับส่งข้อมูลเครือข่าย)

แท็บ Network ใช้สำหรับตรวจสอบการโหลดทรัพยากรต่าง ๆ ของหน้าเว็บ เช่น ไฟล์ HTML, CSS, JavaScript, รูปภาพ รวมถึงการเรียกใช้งาน API (Fetch/XHR):

  1. ติดตามสถานะการโหลด: ตรวจสอบสถานะการโหลด (Status Code เช่น 200 OK, 404 Not Found), เวลาที่ใช้ในการโหลด และขนาดของไฟล์
  2. การกรองข้อมูล (Filtering): สามารถเลือกกรองดูเฉพาะไฟล์รูปภาพ ไฟล์ Script หรือพิมพ์ค้นหาตามชื่อไฟล์ที่ต้องการได้
  3. ดูรายละเอียดเชิงลึก: เมื่อคลิกที่แต่ละรายการ จะสามารถดูข้อมูล Headers, ผลลัพธ์ที่ส่งกลับมาจากเซิร์ฟเวอร์ (Response) และลำดับเวลาในการโหลดได้

💡 ตัวเลือกสำคัญที่ควรรู้:
- Disable Cache: สั่งไม่ให้ Browser ดึงข้อมูลจากแคช เพื่อทดสอบการโหลดหน้าเว็บใหม่จริง ๆ ทุกครั้ง
- Preserve Log: สั่งให้เก็บประวัติการโหลดไว้ ไม่ให้ล้างข้อมูลทิ้งเมื่อมีการรีโหลดหน้าเว็บ
- Throttling: จำลองความเร็วอินเทอร์เน็ต (เช่น 3G ช้า ๆ) เพื่อดูว่าผู้ใช้ที่เน็ตช้าจะใช้งานเว็บของเราได้ดีหรือไม่

5. แท็บ Application (การจัดการข้อมูลที่บันทึกบน Browser)

แท็บ Application ใช้สำหรับตรวจสอบและจัดการข้อมูลที่ถูกบันทึกไว้ใน Browser ของผู้ใช้งาน เช่น ข้อมูลใน Local Storage, Session Storage และ Cookies เราสามารถคลิกเลือกโดเมนเพื่อเข้าไปดู แก้ไขค่า หรือลบข้อมูลเหล่านั้นเพื่อทดสอบระบบล็อกอินหรือการตั้งค่าต่าง ๆ ได้

6. แท็บ Sources (การตรวจสอบไฟล์และการ Debug)

แท็บ Sources แสดงโครงสร้างไฟล์ทั้งหมดที่ถูกโหลดเข้ามาในหน้าเว็บนี้ มีประโยชน์อย่างมากในการเข้าไปดู Source Code และตั้งจุดหยุดทำงานของโปรแกรม (Breakpoints) เพื่อไล่ตรวจหา Bug ใน JavaScript ทีละบรรทัด (ซึ่งจะได้เรียนรู้ในบทเรียนขั้นสูงถัดไป)

7. แท็บ Performance (การวิเคราะห์ความเร็วหน้าเว็บ)

แท็บ Performance ใช้สำหรับบันทึกและวิเคราะห์ความเร็วในการแสดงผลและการตอบสนองของหน้าเว็บ ช่วยระบุจุดที่ทำให้เว็บทำงานช้าหรือกระตุก รวมถึงสามารถจำลองการทำงานบนอุปกรณ์ที่ CPU ช้าลงได้

8. เคล็ดลับการใช้งาน DevTools เพิ่มเติม

  1. เครื่องมือเลือกองค์ประกอบ (Element Picker): คลิกที่ไอคอนลูกศรมุมซ้ายบนสุดของ DevTools จากนั้นนำเมาส์ไปคลิกเลือกองค์ประกอบที่ต้องการบนหน้าจอเพื่อ Inspect ได้ทันที
  2. โหมดจำลองอุปกรณ์มือถือ (Device Mode): คลิกที่ไอคอนรูปแท็บเล็ต/โทรศัพท์เพื่อจำลองขนาดหน้าจอของอุปกรณ์ต่าง ๆ สำหรับทดสอบความยืดหยุ่นของหน้าเว็บ (Responsive Design)
  3. การจัดวางหน้าต่าง (Dock Side): คลิกที่ปุ่มจุดสามจุดของ DevTools เพื่อปรับตำแหน่งการวางหน้าต่างเครื่องมือ เช่น วางไว้ด้านขวา ด้านล่าง ด้านซ้าย หรือแยกหน้าต่างออกมาเป็นอิสระ

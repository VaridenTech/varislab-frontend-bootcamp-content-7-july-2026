พื้นฐาน Browser DevTools

DevTools เป็นเครื่องมือ Debug ที่ทรงพลังในเบราว์เซอร์ ใช้ตรวจสอบ แก้ปัญหา และปรับประสิทธิภาพเว็บ เนื้อหานี้เน้น Chrome แต่แนวคิดใช้กับเบราว์เซอร์อื่นได้

การเปิด DevTools

  1. คีย์ลัด: Windows/Linux ใช้ F12 หรือ Ctrl + Shift + I; Mac ใช้ Cmd + Option + I
  2. คลิกขวา Element แล้วเลือก “Inspect”
  3. เมนู Chrome: จุดสามจุด → More Tools → Developer Tools

ภาพรวม

  1. Elements - ดูและแก้ HTML/CSS
  2. Console - รัน JavaScript และดู Log/Error
  3. Network - ตรวจคำขอเครือข่าย
  4. Application - ดู Storage, Cookie และข้อมูลอื่น
  5. Sources - ดูไฟล์และ Debug ทีละบรรทัด
  6. Performance - วิเคราะห์ประสิทธิภาพ

แท็บ Elements

คุณสมบัติสำคัญ

  1. สำรวจโครงสร้าง HTML แบบลำดับชั้น และชี้เพื่อ Highlight ในหน้า
  2. แก้ HTML แบบ Live ดับเบิลคลิกข้อความหรือคลิกขวาเพื่อแก้ ลบ คัดลอก หรือซ่อน Element
  3. แผง Styles แสดงและแก้ CSS, Computed style, Color picker และ Box model
  4. ดู Event Listener ที่ติดกับ Element
  5. จำลองสถานะ :hover, :active, :focus

ตัวอย่างตรวจ Layout

Inspect Element แล้วตรวจ display: none, visibility: hidden, ขนาดศูนย์ และปัญหา Margin/Padding จากนั้นแก้ค่า CSS โดยตรงเพื่อทดสอบ

ค้นหา CSS ที่ขัดแย้ง

Style ที่ถูกขีดทับคือค่าที่ถูก Override และจะแสดงไฟล์ต้นทางกับเลขบรรทัด

แท็บ Console

เป็นที่แสดง Log/Error และรัน JavaScript โดยตรง:

document.title = "New Title"
document.body.style.backgroundColor = "lightblue"
const button = document.querySelector("button")
button.style.fontSize = "20px"
function add(a, b) { return a + b }
add(5, 3) // 8

หากมีคำเตือนเรื่องการวางข้อมูล ให้พิมพ์ allow pasting แล้วกด Enter

แท็บ Network

แสดงคำขอทั้งหมด เช่น HTML, CSS, JavaScript, รูปภาพ และ API

  1. ติดตาม Request: Status code, เวลา และขนาดไฟล์
  2. กรอง: ตามชนิด Status หรือชื่อไฟล์
  3. รายละเอียด: Headers, Preview, Response และ Timing

ตัวเลือกสำคัญคือ Preserve log, Disable cache และ Throttle

แท็บ Application

ใช้ตรวจข้อมูลที่เบราว์เซอร์เก็บ ได้แก่ Local Storage, Session Storage และ Cookies เลือก Storage กับโดเมนเพื่อดู แก้ หรือลบข้อมูล

แท็บ Sources

แสดงไฟล์ทั้งหมดที่หน้าโหลดและมีประโยชน์สำหรับ Debug JavaScript โดยจะกล่าวถึงอย่างละเอียดในบทถัดไป

แท็บ Performance

บันทึกและวิเคราะห์ประสิทธิภาพขณะทำงาน รวมถึงจำลอง CPU และเครือข่ายที่ช้าลง

เคล็ดลับ DevTools

  1. Element Picker: คลิกไอคอนลูกศรแล้วเลือก Element ในหน้า
  2. Device Simulation: คลิกไอคอนคอมพิวเตอร์/โทรศัพท์เพื่อทดสอบ Responsive
  3. Docking: เลือกวาง DevTools ด้านซ้าย ขวา ล่าง หรือแยกหน้าต่าง

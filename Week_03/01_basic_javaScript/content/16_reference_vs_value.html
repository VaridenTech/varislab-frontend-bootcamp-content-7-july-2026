การส่งค่าแบบอ้างอิง (Reference) vs การส่งค่าแบบข้อมูล (Value)

หนึ่งในแนวคิดที่สำคัญและมักสร้างความสับสนมากที่สุดใน JavaScript คือ วิธีการจัดเก็บและจัดการข้อมูลในหน่วยความจำ (Memory) ของคอมพิวเตอร์ ซึ่งชนิดข้อมูลใน JavaScript จะถูกแบ่งออกเป็น 2 กลุ่มหลักตามพฤติกรรมการเก็บข้อมูล ได้แก่ การเก็บแบบข้อมูล (Value Types / Primitive Types) และ การเก็บแบบอ้างอิง (Reference Types / Objects, Arrays, Functions)

💡 คำแนะนำ: หัวข้อนี้เป็นหนึ่งในรากฐานที่สำคัญที่สุดของการเขียนโปรแกรม แนะนำให้ค่อย ๆ อ่าน ทำความเข้าใจ และลองจินตนาการตามทีละขั้นตอนนะครับ

1. การทำงานของชนิดข้อมูลแบบ Value (Primitive Types)

เมื่อเราประกาศตัวแปรและกำหนดข้อมูลประเภทพื้นฐาน (เช่น Number, String, Boolean) ตัว JavaScript Engine จะนำค่าข้อมูลจริง ๆ ไปจัดเก็บไว้ในกล่องหน่วยความจำของตัวแปรนั้นโดยตรง:

const a = 10
const b = "hi"
ตัวแปร ค่าข้อมูลจริง
a 10
b "hi"

พฤติกรรมการคัดลอกข้อมูลแบบ Value

เมื่อเรานำตัวแปรแบบ Value ไปกำหนดค่าให้กับอีกตัวแปรหนึ่ง ตัว JavaScript จะทำการ คัดลอก (Copy) ค่าข้อมูลจริงสร้างขึ้นมาใหม่อีกชุดหนึ่งแยกต่างหาก ทำให้ตัวแปรทั้งสองไม่มีความเกี่ยวข้องกันหลังจากนั้น:

let a = 10
let b = a // คัดลอกค่าข้อมูลจริงจาก a ไปให้ b

b = b + 1 // การเปลี่ยนแปลงค่า b จะไม่มีผลกระทบใด ๆ ต่อค่าของ a
ตัวแปร ค่าข้อมูลจริง
a 10
b 11

2. การทำงานของชนิดข้อมูลแบบ Reference (Reference Types)

ต่างจากข้อมูลพื้นฐานอย่างสิ้นเชิง! เมื่อเราสร้างข้อมูลประเภท Array, Object หรือ Function ตัว JavaScript Engine จะไม่นำข้อมูลจริง ๆ ไปใส่ในตัวแปรโดยตรง แต่จะนำข้อมูลเหล่านั้นไปเก็บไว้ในพื้นที่ส่วนอื่นของหน่วยความจำ แล้วส่ง "ที่อยู่อ้างอิง" (Memory Address / Reference) กลับมาเก็บไว้ที่ตัวแปรแทน:

const c = [1, 2]
ตัวแปร ที่อยู่อ้างอิง (Reference)
c 0x001
ที่อยู่หน่วยความจำ (Address) ข้อมูลจริงในหน่วยความจำ
0x001 [1, 2]

💡 เปรียบเทียบให้เห็นภาพง่าย ๆ: ให้มองว่าที่อยู่อ้างอิง 0x001 เป็นเหมือน "กุญแจห้องพักโรงแรม" ตัวแปรของเราจะเก็บเพียงแค่กุญแจห้องเท่านั้น ไม่ได้เก็บสิ่งของหรือผู้เข้าพักจริง ๆ ไว้ในตัวแปร เมื่อเราต้องการใช้งานข้อมูล ระบบก็จะใช้กุญแจนี้เพื่อเปิดเข้าไปดูข้อมูลในห้องพักนั้น ๆ

พฤติกรรมการคัดลอกข้อมูลแบบ Reference

เมื่อเรานำตัวแปรประเภท Reference ไปกำหนดค่าให้กับอีกตัวแปรหนึ่ง สิ่งที่ถูกคัดลอกไปคือ "ที่อยู่อ้างอิง" (กุญแจห้อง) ไม่ใช่ข้อมูลจริง ๆ ในหน่วยความจำ:

const c = [1, 2]
const d = c // คัดลอกเฉพาะที่อยู่อ้างอิง (Reference) จาก c ไปให้ d

d.push(3) // การแก้ไขข้อมูลผ่าน d จะส่งผลต่อ c ด้วย เพราะทั้งคู่ชี้ไปยังห้องข้อมูลเดียวกัน
ตัวแปร ที่อยู่อ้างอิง (Reference)
c 0x001
d 0x001
ที่อยู่หน่วยความจำ (Address) ข้อมูลจริงในหน่วยความจำ
0x001 [1, 2, 3]

เนื่องจากทั้ง c และ d ต่างก็เก็บกุญแจห้องเดียวกัน (0x001) การเปลี่ยนแปลงข้อมูลในห้องพักนั้นจึงทำให้ตัวแปรทั้งสองมองเห็นผลลัพธ์ที่เปลี่ยนไปเหมือนกัน

3. การสร้างข้อมูลชุดใหม่ (การจัดสรร Reference ใหม่)

เมื่อใดก็ตามที่เราใช้สัญลักษณ์ [] หรือ {} เพื่อสร้างข้อมูลใหม่ ตัว JavaScript Engine จะทำการเปิดห้องพักใหม่ในหน่วยความจำและจัดสรรที่อยู่อ้างอิงใหม่ให้เสมอ:

let c = [1, 2] // เปิดห้องพักใหม่ ได้ที่อยู่อ้างอิง 0x001
let d = [3, 4, 5] // เปิดห้องพักใหม่อีกห้อง ได้ที่อยู่อ้างอิง 0x002

d.push(6)
ตัวแปร ที่อยู่อ้างอิง (Reference)
c 0x001
d 0x002
ที่อยู่หน่วยความจำ (Address) ข้อมูลจริงในหน่วยความจำ
0x001 [1, 2]
0x002 [3, 4, 5, 6]

4. การเปรียบเทียบข้อมูลประเภท Reference

เนื่องจากตัวดำเนินการเปรียบเทียบ (เช่น == หรือ ===) จะตรวจสอบ "ที่อยู่อ้างอิง" (Reference) ที่ตัวแปรเก็บไว้ ไม่ได้เข้าไปตรวจสอบข้อมูลจริง ๆ ข้างในห้อง ดังนั้น Array หรือ Object สองตัวที่มีข้อมูลเหมือนกันทุกประการ จึงถูกมองว่า "ไม่เท่ากัน" เพราะเก็บกุญแจคนละห้องกัน:

const a = [1, 2]
const b = [1, 2] // ข้อมูลข้างในเหมือนกันเป๊ะ แต่เก็บไว้คนละที่ในหน่วยความจำ!

console.log(a == b) // false
console.log(a === b) // false
ตัวแปร ที่อยู่อ้างอิง (Reference)
a 0x001
b 0x002
ที่อยู่หน่วยความจำ (Address) ข้อมูลจริงในหน่วยความจำ
0x001 [1, 2]
0x002 [1, 2]

เบื้องหลังการทำงาน JavaScript กำลังเปรียบเทียบว่า "ที่อยู่ 0x001 เท่ากับที่อยู่ 0x002 หรือไม่?" คำตอบจึงเป็น false เสมอ แม้ว่าข้อมูลภายในห้องพักทั้งสองจะหน้าตาเหมือนกันก็ตาม

กรณีที่เปรียบเทียบแล้วได้ผลลัพธ์เป็น True

const a = [1, 2]
const b = a // คัดลอกกุญแจห้องเดียวกันมา!

console.log(a === b) // true ← ทั้งสองตัวแปรชี้ไปยังห้องพักข้อมูลเดียวกันในหน่วยความจำ

การทำงานแบบเดียวกันใน Object

let person1 = { name: "Kyle" }
let person2 = person1 // ชี้ไปยังที่อยู่อ้างอิงเดียวกัน

person2.name = "Joe" // เปลี่ยนค่าผ่านตัวแปร person2

console.log(person1.name) // "Joe" ← person1 ก็เปลี่ยนตามเพราะชี้ไปยัง Object เดียวกัน
console.log(person2.name) // "Joe"
console.log(person1 === person2) // true ← เปรียบเทียบแล้วได้ true เพราะเป็นกุญแจห้องเดียวกัน

// แต่ถ้าหากเราสร้าง Object ใหม่ขึ้นมา (เปิดห้องใหม่):
const person3 = { name: "Joe" }
console.log(person1 === person3) // false ← เปรียบเทียบแล้วได้ false เพราะเป็นคนละห้องกัน

5. ความสัมพันธ์ระหว่าง const และ Reference

เมื่อเราประกาศตัวแปรประเภท Reference ด้วย Keyword const ตัวแปรนั้นจะถูกห้ามไม่ให้เปลี่ยน "ที่อยู่อ้างอิง" (ห้ามเปลี่ยนกุญแจห้อง) แต่เรายังคงสามารถเดินเข้าไป แก้ไข ปรับปรุง หรือเพิ่มลดสิ่งของที่อยู่ภายในห้องพักนั้นได้ตามปกติ

จำง่าย ๆ ว่า const จะคอยล็อกไม่ให้เปลี่ยนที่อยู่หน่วยความจำ แต่ไม่ได้ล็อกข้อมูลข้างใน:

const arr = [1, 2]

// ✅ ทำได้: แก้ไขข้อมูลในห้องเดิม (ที่อยู่อ้างอิงคงเดิม)
arr.push(3)
console.log(arr) // [1, 2, 3]

// ❌ ทำไม่ได้: พยายามเปลี่ยนไปชี้ที่อยู่อ้างอิงใหม่ (เปลี่ยนกุญแจห้อง)
arr = [4, 5, 6] // ❌ TypeError: Assignment to constant variable

6. การส่งข้อมูลประเภท Reference เข้าไปใน Function

เมื่อเราส่งข้อมูลประเภท Array หรือ Object เข้าไปเป็น Argument ใน Function สิ่งที่ Function ได้รับไปคือ "ที่อยู่อ้างอิง" (Reference) ดังนั้น หาก Function นั้นมีการแก้ไขข้อมูลใด ๆ ภายใน Array หรือ Object นั้น ข้อมูลต้นฉบับภายนอก Function จะถูกแก้ไขตามไปด้วยทันที:

function addElement(array, element) {
  array.push(element) // แก้ไขข้อมูลใน Array ต้นฉบับโดยตรง!
}

const numbers = [1, 2, 3]
addElement(numbers, 4)

console.log(numbers) // [1, 2, 3, 4] ← Array ต้นฉบับด้านนอกถูกแก้ไขตามไปด้วย

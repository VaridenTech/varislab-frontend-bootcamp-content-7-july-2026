Reference กับ Value

หนึ่งในแนวคิดสำคัญที่สุดของ JavaScript คือวิธีที่ตัวแปรเก็บข้อมูลในหน่วยความจำของคอมพิวเตอร์ ชนิดข้อมูลแบ่งเป็นสองกลุ่มหลัก ได้แก่ ชนิดค่า (primitive) และ ชนิดอ้างอิง (Object, Array และฟังก์ชัน)

⚠️ คำเตือน: หัวข้อนี้เข้าใจยาก จึงควรใช้เวลาทำความเข้าใจให้ถ่องแท้!

ชนิดค่าทำงานอย่างไร

เมื่อกำหนดค่าพื้นฐาน JavaScript จะคัดลอกค่าจริงเข้าไปในหน่วยความจำโดยตรง:

const a = 10
const b = "hi"
ตัวแปรค่า
a10
b"hi"

การคัดลอกชนิดค่า

เมื่อคัดลอกตัวแปรที่เก็บค่าพื้นฐาน JavaScript จะสร้างสำเนาใหม่ของค่านั้นในหน่วยความจำ:

let a = 10
let b = a // คัดลอกค่าของ a

b = b + 1 // เปลี่ยนเฉพาะ b ไม่เปลี่ยน a
ตัวแปรค่า
a10
b11

ชนิดอ้างอิงทำงานอย่างไร

เมื่อสร้าง Array หรือ Object JavaScript จะเก็บ reference (ที่อยู่หน่วยความจำ) ของตำแหน่งข้อมูล:

const c = [1, 2]
ตัวแปรค่า (Reference)
c0x001
ตำแหน่งหน่วยความจำข้อมูลจริง
0x001[1, 2]

ให้นึกถึง 0x001 ว่าเป็น “หมายเลขห้องพักโรงแรม” ตัวแปรเก็บหมายเลขห้อง ไม่ได้เก็บสิ่งของจริงภายในห้อง

การคัดลอกชนิดอ้างอิง

เมื่อคัดลอกตัวแปรที่เก็บ Array หรือ Object JavaScript จะคัดลอก reference ไม่ใช่ข้อมูลจริง:

const c = [1, 2]
const d = c // คัดลอก REFERENCE ไม่ใช่ Array

d.push(3) // แก้ไข Array ที่ทั้ง c และ d ชี้ไป
ตัวแปรค่า (Reference)
c0x001
d0x001
ตำแหน่งหน่วยความจำข้อมูลจริง
0x001[1, 2, 3]

ทั้ง c และ d เก็บที่อยู่หน่วยความจำเดียวกัน (0x001) จึงชี้ไปยัง Array เดียวกัน

การสร้าง Reference ใหม่

เมื่อสร้าง Array หรือ Object ใหม่ JavaScript จะจัดสรรตำแหน่งหน่วยความจำใหม่:

let c = [1, 2] // ตำแหน่งหน่วยความจำ 0x001
let d = [3, 4, 5] // ตำแหน่งหน่วยความจำ 0x002 (ต่างกัน!)

d.push(6)
ตัวแปรค่า (Reference)
c0x001
d0x002
ตำแหน่งหน่วยความจำข้อมูลจริง
0x001[1, 2]
0x002[3, 4, 5, 6]

การเปรียบเทียบ Reference

เนื่องจาก JavaScript เปรียบเทียบค่า Reference ไม่ใช่เนื้อหาจริง Array หรือ Object คนละตัวที่มีเนื้อหาเหมือนกันจึงถือว่าต่างกัน:

const a = [1, 2]
const b = [1, 2] // เนื้อหาเหมือนกัน แต่ตำแหน่งหน่วยความจำต่างกัน!

console.log(a == b) // false
console.log(a === b) // false
ตัวแปรค่า (Reference)
a0x001
b0x002
ตำแหน่งหน่วยความจำข้อมูลจริง
0x001[1, 2]
0x002[1, 2]

JavaScript กำลังถามว่า “0x001 เหมือนกับ 0x002 หรือไม่?” คำตอบคือไม่ แม้เนื้อหาจะเหมือนกัน

เมื่อการเปรียบเทียบคืนค่า True

const a = [1, 2]
const b = a // Reference เดียวกัน!

console.log(a === b) // true ← ทั้งคู่ชี้ไปยังตำแหน่งหน่วยความจำเดียวกัน

Object ทำงานแบบเดียวกัน

let person1 = { name: "Kyle" }
let person2 = person1 // Reference เดียวกัน

person2.name = "Joe"

console.log(person1.name) // "Joe" ← เปลี่ยนแล้ว!
console.log(person2.name) // "Joe" ← Object เดียวกัน
console.log(person1 === person2) // true ← Reference เดียวกัน

// แต่เมื่อสร้าง Object ใหม่:
const person3 = { name: "Joe" }
console.log(person1 === person3) // false ← Reference ต่างกัน

const กับ Reference

คีย์เวิร์ด const ป้องกันการกำหนดตัวแปรใหม่ แต่ไม่ได้ป้องกันการแก้ไขเนื้อหาของ Array หรือ Object ที่ประกาศด้วย const

const สนใจเฉพาะค่า Reference เท่านั้น

const arr = [1, 2]

// ใช้ได้! เราไม่ได้เปลี่ยน Reference
arr.push(3)
console.log(arr) // [1, 2, 3]

// ใช้ไม่ได้! เรากำลังพยายามเปลี่ยน Reference
arr = [4, 5, 6] // ❌ TypeError: Assignment to constant variable
ตัวแปรค่า (Reference)
a0x001

ฟังก์ชันและ Reference

เมื่อส่ง Array หรือ Object ไปยังฟังก์ชัน คุณกำลังส่ง Reference ซึ่งหมายความว่าฟังก์ชันสามารถแก้ไขข้อมูลต้นฉบับได้:

function addElement(array, element) {
  array.push(element) // แก้ไข Array ต้นฉบับ!
}

const numbers = [1, 2, 3]
addElement(numbers, 4)

console.log(numbers) // [1, 2, 3, 4] ← Array ต้นฉบับถูกแก้ไข

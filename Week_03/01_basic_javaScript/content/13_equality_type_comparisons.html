การเปรียบเทียบความเท่ากัน: == และ ===

ในภาษา JavaScript มีวิธีการเปรียบเทียบความเท่ากันอยู่ 2 รูปแบบหลัก ๆ ซึ่งเบื้องหลังการทำงานมีความแตกต่างกันอย่างมากและอาจส่งผลต่อผลลัพธ์ของโปรแกรมได้

1. เครื่องหมายเท่ากับสองตัว (==) - Loose Equality (มีการแปลงชนิดข้อมูล)

ตัวดำเนินการ == จะทำการแปลงชนิดข้อมูลของค่าทั้งสองฝั่งให้เป็นประเภทเดียวกันก่อน (Type Coercion) แล้วจึงทำการเปรียบเทียบค่า:

console.log(1 == "1") // true (จริง - แปลงข้อความ "1" เป็นตัวเลข 1 ก่อนเปรียบเทียบ)

console.log(0 == false) // true (จริง - 0 มีสถานะเป็น Falsy Value)
console.log("" == false) // true (จริง - ข้อความว่างมีสถานะเป็น Falsy Value)

ทำไมจึงเป็นเช่นนั้น? นั่นเป็นเพราะ JavaScript Engine จะแปลงค่าเหล่านี้ให้กลายเป็นข้อมูลชนิดเดียวกันก่อนเปรียบเทียบ โดยตัวเลข 0 และข้อความว่าง "" มีสถานะเป็นค่า "Falsy" (ค่าที่เทียบเท่ากับความเท็จ) ซึ่งระบบจะมองว่ามีค่าเท่ากับ false นั่นเอง

2. เครื่องหมายเท่ากับสามตัว (===) - Strict Equality (ไม่มีการแปลงชนิดข้อมูล)

ตัวดำเนินการ === จะเปรียบเทียบทั้ง ค่าข้อมูล (Value) และ ชนิดข้อมูล (Type) ไปพร้อม ๆ กัน โดยไม่มีการแปลงชนิดข้อมูลใด ๆ เบื้องหลัง:

console.log(1 === 1) // true (จริง - ชนิดข้อมูลและค่าเหมือนกันทุกประการ)
console.log(1 === "1") // false (เท็จ - ชนิดข้อมูลต่างกัน แม้ค่าจะดูเหมือนกัน)
console.log(0 === false) // false (เท็จ - ชนิดข้อมูลต่างกัน)
console.log("" === false) // false (เท็จ - ชนิดข้อมูลต่างกัน)

3. การเปรียบเทียบความไม่เท่ากัน: != และ !==

กฎการแปลงชนิดข้อมูลแบบเดียวกันนี้จะถูกนำมาใช้กับการเปรียบเทียบความไม่เท่ากันเช่นกัน:

console.log(1 != "1") // false (เท็จ - แปลงชนิดข้อมูลแล้วพบว่ามีค่าเท่ากัน)
console.log(1 !== "1") // true (จริง - ชนิดข้อมูลต่างกัน จึงถือว่าไม่เท่ากันอย่างชัดเจน)

สรุปแนวทางปฏิบัติที่ดี (Best Practices)

ในชีวิตการทำงานจริง แนะนำให้เลือกใช้ === และ !== เป็นหลักเสมอ เพื่อป้องกันปัญหาและ Bug ที่คาดไม่ถึงจากการแปลงชนิดข้อมูลอัตโนมัติของ JavaScript

ข้อยกเว้นที่ยอมรับได้: การตรวจสอบ null และ undefined

มีเพียงกรณีเดียวที่เหล่านักพัฒนานิยมใช้ == คือเมื่อต้องการตรวจสอบว่าตัวแปรนั้นมีค่าเป็น null หรือ undefined หรือไม่พร้อม ๆ กันในคำสั่งเดียว:

console.log(null == null) // true
console.log(null == undefined) // true (จริง - ในการเปรียบเทียบแบบหลวม null และ undefined จะถูกมองว่าเท่ากัน)

console.log(null === null) // true
console.log(null === undefined) // false (ชนิดต่างกัน)

เมื่อเราต้องการตรวจสอบว่าตัวแปรหนึ่ง ๆ มีข้อมูลอยู่จริงหรือไม่ การใช้ variable == null จะช่วยตรวจจับได้ทั้งกรณีที่ตัวแปรเป็น null และ undefined ได้พร้อมกันอย่างกระชับและสะดวกมาก

ความเท่ากัน: == กับ ===

JavaScript มีสองวิธีในการตรวจสอบความเท่ากัน และทั้งสองทำงานแตกต่างกันมาก

เครื่องหมายเท่ากับสองตัว == - มีการแปลงชนิดข้อมูล

ตัวดำเนินการ == จะแปลงชนิดข้อมูลก่อนเปรียบเทียบ:

console.log(1 == "1") // true (แปลง String "1" เป็น Number 1)

console.log(0 == false) // true (0 เป็นค่า "falsy")
console.log("" == false) // true (String ว่างเป็นค่า "falsy")

ทำไม? JavaScript แปลงค่าเหล่านี้ให้เป็นชนิดเดียวกันก่อนเปรียบเทียบ เลขศูนย์และ String ว่างถือเป็นค่า “falsy” ที่เท่ากับ false

เครื่องหมายเท่ากับสามตัว === - ไม่มีการแปลงชนิดข้อมูล

ตัวดำเนินการ === เปรียบเทียบทั้งค่า และ ชนิด โดยไม่แปลงชนิดข้อมูล:

console.log(1 === 1) // true (ชนิดและค่าเหมือนกัน)
console.log(1 === "1") // false (ชนิดต่างกัน)
console.log(0 === false) // false (ชนิดต่างกัน)
console.log("" === false) // false (ชนิดต่างกัน)

ไม่เท่ากัน: != กับ !==

กฎเดียวกันนี้ใช้กับการเปรียบเทียบว่าไม่เท่ากัน:

console.log(1 != "1") // false (แปลงชนิดแล้วจึงเปรียบเทียบ)
console.log(1 !== "1") // true (ชนิดต่างกัน จึงไม่เท่ากัน)

ควรใช้แบบใด?

เลือกใช้ === และ !== เสมอ เพื่อหลีกเลี่ยงปัญหาจากการแปลงชนิดข้อมูลที่ไม่คาดคิด

ข้อยกเว้น: null และ undefined

มีกรณีหนึ่งที่ == มีประโยชน์ คือการตรวจสอบ null หรือ undefined:

console.log(null == null) // true
console.log(null == undefined) // true (ถือว่า null และ undefined เท่ากัน)

console.log(null === null) // true
console.log(null === undefined) // false (ชนิดต่างกัน)

เมื่อตรวจสอบว่าตัวแปรมีค่าหรือไม่ คุณมักต้องการตรวจพบทั้ง null และ undefined นี่คือเหตุผลที่ == มีประโยชน์ในกรณีนี้

คำสั่ง Switch

คำสั่ง Switch เป็นวิธีที่สะอาดกว่าในการเขียนเงื่อนไข if-else หลายชุด เมื่อเปรียบเทียบตัวแปรหนึ่งตัวกับค่าหลายค่า

ไวยากรณ์ Switch พื้นฐาน

const favoriteAnimal = "cat"

switch (favoriteAnimal) {
  case "cat":
    console.log("Cats are pretty cool")
    break
  case "dog":
    console.log("They are kinda loud")
    break
  case "shark":
    console.log("That is an interesting choice")
    break
  default:
    console.log("That is cool but I am unfamiliar with that animal")
}

ทำงานอย่างไร

  1. นิพจน์ Switch: ตัวแปรที่ต้องตรวจสอบ (favoriteAnimal)
  2. ป้าย Case: แต่ละ case ตรวจว่านิพจน์ Switch เท่ากับค่าหนึ่งหรือไม่
  3. คำสั่ง Break: จบบล็อก Switch เมื่อพบค่าที่ตรงกัน
  4. Default Case: ทำงานเมื่อไม่มี Case ใดตรงกัน คล้าย else

Switch กับ If-Else

คำสั่ง Switch ข้างต้นเทียบเท่ากับ if-else ชุดนี้:

const favoriteAnimal = "cat"

if (favoriteAnimal === "cat") {
  console.log("Cats are pretty cool")
} else if (favoriteAnimal === "dog") {
  console.log("They are kinda loud")
} else if (favoriteAnimal === "shark") {
  console.log("That is an interesting choice")
} else {
  console.log("That is cool but I am unfamiliar with that animal")
}

จะเห็นว่า Switch ช่วยตัดการตรวจ favoriteAnimal === ที่ซ้ำกันออก

เมื่อใดควรใช้ Switch หรือ If-Else

ใช้ Switch เมื่อ:

  • เปรียบเทียบตัวแปรหนึ่งตัวกับค่าที่เจาะจงหลายค่า
  • มีหลายเงื่อนไขให้ตรวจ
  • เงื่อนไขใช้ strict equality (===)

ใช้ If-Else เมื่อ:

  • ต้องใช้เงื่อนไขซับซ้อน (>, <, &&, ||)
  • เปรียบเทียบตัวแปรต่างตัวกัน
  • มีเงื่อนไขเพียงไม่กี่ข้อ

คุณสมบัติขั้นสูง

หลาย Case สำหรับการทำงานเดียวกัน

หากต้องการให้หลาย Case รันโค้ดเดียวกัน ให้วางต่อกันโดยไม่มี Break คั่น:

const day = "saturday"

switch (day) {
  case "monday":
  case "tuesday":
  case "wednesday":
  case "thursday":
  case "friday":
    console.log("Workday")
    break
  case "saturday":
  case "sunday":
    console.log("Weekend!") // บรรทัดนี้ทำงาน
    break
  default:
    console.log("Invalid day")
}

ทำงานได้เพราะ Switch จะรันโค้ดต่อไปจนกว่าจะพบคำสั่ง Break

การสร้าง Scope ด้วยวงเล็บปีกกา

สำคัญ: แต่ละ Case ไม่ได้สร้าง Scope ของตัวเองโดยค่าเริ่มต้น หากต้องประกาศตัวแปรใน Case ให้ครอบด้วยวงเล็บปีกกา:

const action = "login"

switch (action) {
  case "login": {
    // วงเล็บปีกกาสร้าง Scope ใหม่
    const message = "Welcome back!"
    const timestamp = new Date()
    console.log(`${message} Login time: ${timestamp}`)
    break
  }
  case "logout": {
    // Scope นี้แยกจาก Case login
    const message = "Goodbye!"
    const timestamp = new Date()
    console.log(`${message} Logout time: ${timestamp}`)
    break
  }
  case "register": {
    const message = "Account created!"
    console.log(message)
    break
  }
  default:
    console.log("Unknown action")
}

ทำไม Scope จึงสำคัญ

หากไม่มีวงเล็บปีกกา คุณไม่สามารถประกาศตัวแปรซ้ำได้:

const status = "success"

switch (status) {
  case "success":
    const message = "Operation successful!" // ประกาศ message
    console.log(message)
    break
  case "error":
    const message = "Operation failed!" // ❌ Error! ประกาศ 'message' ซ้ำไม่ได้
    console.log(message)
    break
}

เมื่อมีวงเล็บปีกกา แต่ละ Case จะมี Scope ของตัวเอง:

const status = "success"

switch (status) {
  case "success": {
    const message = "Operation successful!" // ✅ ใช้ได้
    console.log(message)
    break
  }
  case "error": {
    const message = "Operation failed!" // ✅ ใช้ได้ - คนละ Scope
    console.log(message)
    break
  }
}

ข้อผิดพลาดที่พบบ่อย

  1. ลืมคำสั่ง Break

    หากลืม break โค้ดจะ “ไหลผ่าน” ไปยัง Case ถัดไป:

    const fruit = "apple"

switch (fruit) {
  case "apple":
    console.log("Apple selected")
  // ไม่มี Break จึงไหลผ่าน
  case "banana":
    console.log("Banana selected") // บรรทัดนี้ทำงานด้วย!
    break
  default:
    console.log("Unknown fruit")
}

  2. พึ่งพา Loose Equality

    Switch ใช้ strict equality (===) จึงต้องระวังชนิดข้อมูล:

    const value = "5"

switch (value) {
  case 5: // ❌ ไม่ตรงกัน เพราะ 5 เป็น Number ไม่ใช่ String
    console.log("Matched number 5")
    break
  default:
    console.log("No match") // บรรทัดนี้ทำงาน
}

  3. ลืมสร้าง Scope ให้ตัวแปร

    หากประกาศตัวแปรใน Case โดยไม่มีวงเล็บปีกกา ตัวแปรจะรั่วไปยัง Scope ของ Switch:

    const action = "login"

switch (action) {
  case "login":
    const message = "Welcome!" // รั่วไปยัง Scope ของ Switch
    console.log(message)
    break
  case "logout":
    console.log(message) // ❌ Error - message ไม่ได้กำหนดไว้ตรงนี้
    break
}

    ควรใช้วงเล็บปีกกาเพื่อสร้าง Scope ใหม่เสมอเมื่อประกาศตัวแปรใน Case

แบบฝึกหัด

สร้างคำสั่ง Switch ที่รับหมายเลขเดือน (1–12) และส่งฤดูกลับมา:

  • ธันวาคม (12), มกราคม (1), กุมภาพันธ์ (2): “Winter”
  • มีนาคม (3), เมษายน (4), พฤษภาคม (5): “Spring”
  • มิถุนายน (6), กรกฎาคม (7), สิงหาคม (8): “Summer”
  • กันยายน (9), ตุลาคม (10), พฤศจิกายน (11): “Fall”
เฉลย
function getSeason(month) {
  switch (month) {
    case 12:
    case 1:
    case 2:
      return "Winter"
    case 3:
    case 4:
    case 5:
      return "Spring"
    case 6:
    case 7:
    case 8:
      return "Summer"
    case 9:
    case 10:
    case 11:
      return "Fall"
    default:
      return "Invalid month"
  }
}

console.log(getSeason(7)) // "Summer"
console.log(getSeason(13)) // "Invalid month"

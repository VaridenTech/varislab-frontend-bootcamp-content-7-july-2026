การใช้งานคำสั่ง Switch

คำสั่ง Switch (Switch Statement) เป็นรูปแบบการเขียนเงื่อนไขทางเลือกที่มีประสิทธิภาพและสะอาดตามากกว่าการเขียน if-else if-else หลาย ๆ ชุดซ้อนกัน เหมาะสำหรับกรณีที่เราต้องการเปรียบเทียบค่าของตัวแปรตัวหนึ่งกับตัวเลือกหลาย ๆ ตัวเลือก

1. โครงสร้างไวยากรณ์ Switch พื้นฐาน

const favoriteAnimal = "cat"

switch (favoriteAnimal) {
  case "cat":
    console.log("Cats are pretty cool")
    break
  case "dog":
    console.log("They are kinda loud")
    break
  case "shark":
    console.log("That is an interesting choice")
    break
  default:
    console.log("That is cool but I am unfamiliar with that animal")
}

กลไกการทำงานของ Switch

  1. นิพจน์ Switch (Switch Expression): ตัวแปรหรือค่าที่เราต้องการนำมาตรวจสอบ (ในตัวอย่างคือ favoriteAnimal)
  2. ป้าย Case (Case Label): ตัวเลือกแต่ละตัว (case) ที่จะนำค่ามาตรวจสอบว่าตรงกับนิพจน์ Switch หรือไม่
  3. คำสั่ง Break (break): คำสั่งที่ใช้สำหรับหยุดและกระโดดออกจากบล็อก Switch ทันทีเมื่อทำงานใน Case นั้นเสร็จสิ้น
  4. Default Case (default): ชุดคำสั่งที่จะทำงานในกรณีที่ไม่มี Case ใดตรงกับค่าที่นำมาเปรียบเทียบเลย (ทำหน้าที่เหมือนกับ else ในคำสั่งเงื่อนไขปกติ)

การเปรียบเทียบระหว่าง Switch และ If-Else

หากเราเขียนโค้ดการทำงานของ Switch ด้านบนให้อยู่ในรูปของ if-else จะได้หน้าตาแบบนี้:

const favoriteAnimal = "cat"

if (favoriteAnimal === "cat") {
  console.log("Cats are pretty cool")
} else if (favoriteAnimal === "dog") {
  console.log("They are kinda loud")
} else if (favoriteAnimal === "shark") {
  console.log("That is an interesting choice")
} else {
  console.log("That is cool but I am unfamiliar with that animal")
}

จะเห็นได้ว่าการใช้ switch ช่วยลดความซ้ำซ้อนในการเขียน favoriteAnimal === ซ้ำ ๆ หลายรอบ ทำให้โค้ดดูสะอาดและเป็นระเบียบขึ้นมาก

2. เมื่อใดควรใช้ Switch หรือ If-Else?

ควรเลือกใช้ Switch เมื่อ:

  • ต้องการเปรียบเทียบค่าของตัวแปรตัวเดียวกับค่าที่ระบุเจาะจงหลาย ๆ ค่า
  • มีทางเลือกหรือเงื่อนไขจำนวนมาก
  • การเปรียบเทียบเงื่อนไขทั้งหมดใช้เครื่องหมายเท่ากับแบบเข้มงวด (Strict Equality - ===)

ควรเลือกใช้ If-Else เมื่อ:

  • ต้องใช้การเปรียบเทียบเงื่อนไขที่ซับซ้อน (เช่น การใช้เครื่องหมาย >, <, &&, ||)
  • ต้องการเปรียบเทียบตัวแปรคนละตัวกันในแต่ละเงื่อนไข
  • มีทางเลือกหรือเงื่อนไขเพียงไม่กี่ข้อ

3. ฟีเจอร์และเทคนิคขั้นสูงของ Switch

การรวมหลาย Case เพื่อทำงานร่วมกัน (Multi-case grouping)

หากเราต้องการให้มีตัวเลือกหลายตัวรันชุดคำสั่งเดียวกัน เราสามารถเขียน case วางเรียงต่อกันโดยไม่ต้องใส่คำสั่ง break คั่นกลางได้เลย:

const day = "saturday"

switch (day) {
  case "monday":
  case "tuesday":
  case "wednesday":
  case "thursday":
  case "friday":
    console.log("Workday")
    break
  case "saturday":
  case "sunday":
    console.log("Weekend!") // บรรทัดนี้ทำงาน
    break
  default:
    console.log("Invalid day")
}

กลไกนี้ทำงานได้เนื่องจากเมื่อ JavaScript พบ Case ที่ตรงกันแล้ว มันจะทำงานและไหลผ่าน (Fall-through) ไปยัง Case ถัดไปเรื่อย ๆ จนกว่าจะพบคำสั่ง break

การสร้างขอบเขตตัวแปร (Scope) ด้วยวงเล็บปีกกา {}

⚠️ กฎสำคัญมาก: โดยปกติแล้ว แต่ละ case ภายใน Switch จะไม่ได้สร้างขอบเขตตัวแปร (Scope) ของตัวเองขึ้นมาใหม่ หมายความว่าหากเราประกาศตัวแปรใน Case หนึ่ง ตัวแปรนั้นจะรั่วไหลออกไปและอาจชนกับ Case อื่นได้ หากต้องการประกาศตัวแปรเฉพาะเจาะจงในแต่ละ Case เราจำเป็นต้องครอบชุดคำสั่งใน Case นั้นด้วยวงเล็บปีกกา {}:

const action = "login"

switch (action) {
  case "login": {
    // วงเล็บปีกกาช่วยสร้าง Scope ใหม่สำหรับ Case นี้
    const message = "Welcome back!"
    const timestamp = new Date()
    console.log(`${message} Login time: ${timestamp}`)
    break
  }
  case "logout": {
    // Scope นี้จะถูกแยกเป็นอิสระจาก Case login
    const message = "Goodbye!"
    const timestamp = new Date()
    console.log(`${message} Logout time: ${timestamp}`)
    break
  }
  case "register": {
    const message = "Account created!"
    console.log(message)
    break
  }
  default:
    console.log("Unknown action")
}

ทำไมการสร้าง Scope จึงมีความสำคัญ?

หากเราไม่ใส่วงเล็บปีกกา {} ครอบแต่ละ Case ระบบจะมองว่าตัวแปรเหล่านั้นอยู่ในขอบเขต (Scope) เดียวกันทั้งหมด ส่งผลให้เราไม่สามารถประกาศตัวแปรชื่อซ้ำกันได้ และจะเกิดข้อผิดพลาดขึ้น:

const status = "success"

switch (status) {
  case "success":
    const message = "Operation successful!" // ประกาศตัวแปร message
    console.log(message)
    break
  case "error":
    const message = "Operation failed!" // ❌ Error! ไม่สามารถประกาศตัวแปร message ซ้ำใน Scope เดียวกันได้
    console.log(message)
    break
}

แต่เมื่อเราใช้วงเล็บปีกกา {} ครอบแต่ละ Case จะทำให้แต่ละ Case มี Scope แยกจากกันเป็นอิสระ ทำให้เราสามารถใช้ชื่อตัวแปรซ้ำกันได้โดยไม่เกิดข้อผิดพลาด:

const status = "success"

switch (status) {
  case "success": {
    const message = "Operation successful!" // ✅ ทำงานได้ปกติ
    console.log(message)
    break
  }
  case "error": {
    const message = "Operation failed!" // ✅ ทำงานได้ปกติเพราะอยู่คนละ Scope
    console.log(message)
    break
  }
}

4. ข้อผิดพลาดที่พบบ่อย (Common Pitfalls)

  1. การลืมใส่คำสั่ง Break

    หากเราลืมเขียนคำสั่ง break ปิดท้าย Case โค้ดจะทำงานไหลผ่าน (Fall-through) ไปรันคำสั่งของ Case ถัดไปเรื่อย ๆ แม้ว่าเงื่อนไขของ Case ถัดไปจะไม่ตรงก็ตาม:

    const fruit = "apple"

switch (fruit) {
  case "apple":
    console.log("Apple selected")
    // ⚠️ ไม่มี break ปิดท้าย ทำให้โค้ดไหลผ่าน
  case "banana":
    console.log("Banana selected") // ⚠️ บรรทัดนี้จะทำงานด้วยแม้ว่าผลไม้จะไม่ใช่ banana!
    break
  default:
    console.log("Unknown fruit")
}

  2. ความเข้าใจผิดเรื่องการเปรียบเทียบค่า (Loose Equality)

    คำสั่ง Switch จะใช้การเปรียบเทียบแบบเข้มงวด (Strict Equality - ===) เสมอ ดังนั้นชนิดข้อมูล (Data Type) จะต้องตรงกันทุกประการ มิฉะนั้นจะไม่พบ Case ที่ตรงกันเลย:

    const value = "5"

switch (value) {
  case 5: // ❌ ไม่ตรงกัน เพราะ 5 เป็น Number แต่ตัวแปร value เป็น String "5"
    console.log("Matched number 5")
    break
  default:
    console.log("No match") // บรรทัดนี้ทำงาน
}

  3. การประกาศตัวแปรโดยไม่มี Scope ครอบ

    หากเราประกาศตัวแปรภายใน Case โดยไม่มีวงเล็บปีกกา {} ครอบ ตัวแปรนั้นจะรั่วไหลและอาจทำให้เกิดข้อผิดพลาดเมื่อเรียกใช้ใน Case อื่น:

    const action = "login"

switch (action) {
  case "login":
    const message = "Welcome!" // ตัวแปร message จะรั่วไหลไปยัง Scope หลักของ Switch
    console.log(message)
    break
  case "logout":
    console.log(message) // ❌ Error - เนื่องจาก message ไม่ได้ถูกกำหนดค่าไว้สำหรับกรณี logout
    break
}

    ดังนั้น ควรสร้าง Scope แยกสำหรับแต่ละ Case ด้วยวงเล็บปีกกา {} ทุกครั้งที่มีการประกาศตัวแปรภายใน Case

ลองทำแบบฝึกหัด

จงเขียนคำสั่ง Switch ที่รับหมายเลขเดือน (1-12) แล้วคืนค่าเป็นชื่อฤดูกาล (Season) ตามเงื่อนไขดังนี้:

  • ธันวาคม (12), มกราคม (1), กุมภาพันธ์ (2): “Winter”
  • มีนาคม (3), เมษายน (4), พฤษภาคม (5): “Spring”
  • มิถุนายน (6), กรกฎาคม (7), สิงหาคม (8): “Summer”
  • กันยายน (9), ตุลาคม (10), พฤศจิกายน (11): “Fall”
ดูเฉลย 
function getSeason(month) {
  switch (month) {
    case 12:
    case 1:
    case 2:
      return "Winter"
    case 3:
    case 4:
    case 5:
      return "Spring"
    case 6:
    case 7:
    case 8:
      return "Summer"
    case 9:
    case 10:
    case 11:
      return "Fall"
    default:
      return "Invalid month"
  }
}

console.log(getSeason(7)) // "Summer"
console.log(getSeason(13)) // "Invalid month"

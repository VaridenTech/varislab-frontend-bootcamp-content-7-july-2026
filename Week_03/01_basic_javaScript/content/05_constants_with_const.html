const - การสร้างค่าคงที่

const ใช้สร้างตัวแปรที่เป็น ค่าคงที่ ซึ่งค่าของมันจะไม่มีวันเปลี่ยนแปลง

const คืออะไร?

const สร้างตัวแปรที่ไม่สามารถกำหนดค่าใหม่ได้หลังจากสร้างแล้ว ลองนึกถึงปากกาเคมี เมื่อคุณกำหนดค่าแล้ว ค่านั้นจะไม่สามารถเปลี่ยนแปลงได้

const name = "Alice"

// การพยายามเปลี่ยนตัวแปร const จะทำให้เกิดข้อผิดพลาด
name = "Bob" // ❌ TypeError: Assignment to constant variable

ความแตกต่างจาก let

  1. ต้องกำหนดค่าเมื่อสร้าง

    ต่างจาก let ตรงที่คุณไม่สามารถสร้างตัวแปร const โดยไม่กำหนดค่าให้มันได้:

    // ✅ ถูกต้อง - const ที่มีค่าเริ่มต้น
const name = "Alice"
const age = 45

const message // ❌ SyntaxError: Missing initializer in const declaration

  2. ไม่สามารถกำหนดค่าใหม่ได้

    เมื่อสร้างแล้ว คุณไม่สามารถเปลี่ยนค่าของตัวแปร const ได้:

    const name = "Alice"

name = "Bob" // ❌ TypeError: Assignment to constant variable

แนวทางตัดสินใจเลือกระหว่าง const กับ let

ผมใช้ const เป็นค่าเริ่มต้น เว้นแต่จะรู้ว่าค่านั้นจะเปลี่ยนแปลง วิธีนี้ช่วยป้องกันการเปลี่ยนค่าที่สำคัญโดยไม่ตั้งใจ

ใช้ const เมื่อ:

  • ค่าจะไม่มีวันเปลี่ยนแปลง
  • เป็นค่าคงที่ส่วนกลาง เช่น การกำหนดค่าหรือการตั้งค่า

ใช้ let เมื่อ:

  • ค่าจะเปลี่ยนแปลงในขณะที่โปรแกรมทำงาน
  • คุณกำลังนับหรือติดตามบางสิ่ง

ค่าคงที่ส่วนกลาง

ในแอปพลิเคชันขนาดใหญ่ คุณอาจกำหนดค่าคงที่ส่วนกลางที่นำไปใช้ทั่วทั้งโค้ดเบส:

const APP_NAME = "My Awesome App"
const VERSION = "1.0.0"
const SUPPORT_EMAIL = "support@example.com"

ค่าคงที่ส่วนกลางเหล่านี้มักเขียนด้วยตัวพิมพ์ใหญ่และใช้ขีดล่างคั่นระหว่างคำ

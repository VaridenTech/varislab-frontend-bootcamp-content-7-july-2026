การสร้างค่าคงที่ด้วย const

const ใช้สำหรับสร้างตัวแปรที่เป็น ค่าคงที่ (Constants) ซึ่งเมื่อกำหนดค่าแล้วจะไม่สามารถเปลี่ยนแปลงหรือกำหนดค่าใหม่ได้

const คืออะไร?

const ใช้สร้างตัวแปรที่ไม่สามารถเปลี่ยนค่าใหม่ได้หลังจากประกาศแล้ว เปรียบเสมือนการเขียนด้วยปากกาเคมีที่ลบไม่ได้ เมื่อเขียนลงไปแล้วจะไม่สามารถแก้ไขได้

const name = "Alice"

// การพยายามเปลี่ยนตัวแปร const จะทำให้เกิดข้อผิดพลาด
name = "Bob" // ❌ TypeError: Assignment to constant variable (ไม่สามารถกำหนดค่าใหม่ให้ตัวแปร const ได้)

ความแตกต่างระหว่าง const และ let

  1. ต้องกำหนดค่าเริ่มต้นทันทีเมื่อประกาศตัวแปร

    ต่างจาก let ตรงที่คุณไม่สามารถประกาศตัวแปร const ทิ้งไว้เฉย ๆ โดยไม่มีการกำหนดค่าเริ่มต้นได้:

    // ✅ ถูกต้อง - กำหนดค่าเริ่มต้นทันที
const name = "Alice"
const age = 45

const message // ❌ SyntaxError: Missing initializer in const declaration

  2. ไม่สามารถกำหนดค่าใหม่ (Reassign) ได้

    เมื่อกำหนดค่าเริ่มต้นแล้ว คุณจะไม่สามารถเปลี่ยนแปลงหรือกำหนดค่าใหม่ให้กับตัวแปร const ได้อีก:

    const name = "Alice"

name = "Bob" // ❌ TypeError: Assignment to constant variable

แนวทางการเลือกระหว่าง const และ let

แนวทางปฏิบัติที่ดีคือแนะนำให้ใช้ const เป็นค่าเริ่มต้นเสมอ เว้นแต่คุณจะแน่ใจว่าค่าของตัวแปรนั้นจำเป็นต้องเปลี่ยนแปลงในภายหลัง วิธีนี้จะช่วยป้องกันการเผลอเปลี่ยนค่าสำคัญโดยไม่ได้ตั้งใจ

ควรใช้ const เมื่อ:

  • ค่าของข้อมูลไม่มีการเปลี่ยนแปลงตลอดการทำงานของโปรแกรม
  • เป็นค่าคงที่ส่วนกลาง (Global Constants) เช่น การตั้งค่าระบบ หรือค่าคงที่ทางคณิตศาสตร์

ควรใช้ let เมื่อ:

  • ค่าของข้อมูลมีการเปลี่ยนแปลงระหว่างการทำงานของโปรแกรม
  • ใช้สำหรับนับรอบ หรือติดตามสถานะที่มีการอัปเดตค่าบ่อย ๆ (เช่น ในลูป)

การใช้งานค่าคงที่ส่วนกลาง (Global Constants)

ในโปรเจกต์ขนาดใหญ่ นิยมประกาศค่าคงที่ส่วนกลางไว้ที่ด้านบนสุดของไฟล์หรือแยกไฟล์ไว้ เพื่อนำไปเรียกใช้ในจุดต่าง ๆ ของโปรแกรม:

const APP_NAME = "My Awesome App"
const VERSION = "1.0.0"
const SUPPORT_EMAIL = "support@example.com"

ตามธรรมเนียมปฏิบัติ (Convention) ค่าคงที่ส่วนกลางเหล่านี้มักจะเขียนด้วย ตัวพิมพ์ใหญ่ทั้งหมด (UPPERCASE) และใช้เครื่องหมายขีดล่าง (_) คั่นระหว่างคำ (เรียกว่า SCREAMING_SNAKE_CASE)

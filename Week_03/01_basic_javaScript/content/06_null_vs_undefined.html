การแสดงถึง “การไม่มีค่า”

การแสดงถึงการไม่มีค่าใน JavaScript นั้นชวนให้สับสน เพราะมีสองวิธีที่แตกต่างกันในการทำเช่นนี้ ได้แก่ null และ undefined

null และ undefined คืออะไร?

ทั้ง null และ undefined แสดงถึงการไม่มีค่า แต่ถูกใช้ในสถานการณ์ที่แตกต่างกัน:

  • undefined - หมายความว่ามีตัวแปรอยู่ แต่ยังไม่ได้รับการกำหนดค่า
  • null - หมายความว่าคุณตั้งใจให้ตัวแปรไม่มีค่า

ทำความเข้าใจ undefined

เมื่อคุณสร้างตัวแปรแต่ไม่ได้กำหนดค่าให้ JavaScript จะตั้งค่าให้เป็น undefined โดยอัตโนมัติ:

let a
console.log(a) // undefined

ยังมีกรณีอื่น ๆ ที่คุณอาจพบ undefined ซึ่งจะกล่าวถึงในภายหลัง แต่ค่านี้หมายความเสมอว่าไม่มีค่าอยู่

ทำความเข้าใจ null

null เป็นค่าที่ใช้ระบุว่า “ตั้งใจให้ว่าง”:

let a = null
console.log(a) // null

การกำหนดค่าเป็น null คือการระบุอย่างชัดเจนว่าตัวแปรนี้ ไม่มีค่า

แนวทางปฏิบัติที่ดี

  1. ใช้ null สำหรับสถานะ “ว่าง” โดยตั้งใจ

    let selectedFile = null
let currentTheme = null
let lastError = null

  2. ปล่อยให้ undefined เกิดขึ้นเองตามธรรมชาติ

    let userName // จะกำหนดค่าในภายหลัง
let userPreferences // จะโหลดจากเซิร์ฟเวอร์

  3. ใช้ null เพื่อล้างค่า

    let name = "Kyle"
// โค้ดอื่นใช้ name...
name = null // ตอนนี้ name ไม่มีค่า

ความแตกต่างที่สำคัญ

  • undefined - ฉันยังไม่ได้รับการกำหนดค่า
  • null - ฉันไม่มีค่าโดยตั้งใจ

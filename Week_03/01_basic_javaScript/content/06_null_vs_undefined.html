Null vs Undefined: การแสดงถึง “การไม่มีค่า”

การแสดงถึงการไม่มีค่าใน JavaScript อาจทำให้ผู้เริ่มต้นสับสนได้ง่าย เนื่องจากมีสองคำที่มีความหมายคล้ายกันแต่ใช้งานต่างกัน นั่นคือ null และ undefined

null และ undefined คืออะไร?

แม้ว่าทั้ง null และ undefined จะหมายถึงการไม่มีข้อมูลเหมือนกัน แต่ถูกนำมาใช้ในวัตถุประสงค์ที่ต่างกันอย่างชัดเจน:

  • undefined - หมายความว่า มีการประกาศตัวแปรไว้แล้ว แต่ยังไม่ได้ถูกกำหนดค่าใด ๆ ให้กับตัวแปรนั้น (เกิดขึ้นโดยอัตโนมัติจากระบบ)
  • null - หมายความว่า ผู้เขียนโปรแกรมตั้งใจกำหนดให้ตัวแปรนั้นไม่มีค่าหรือว่างเปล่า

ทำความเข้าใจ undefined

เมื่อคุณประกาศตัวแปรแต่ไม่ได้กำหนดค่าเริ่มต้นให้มัน JavaScript Engine จะกำหนดค่าเริ่มต้นให้เป็น undefined โดยอัตโนมัติ:

let a
console.log(a) // undefined

นอกจากนี้ยังมีกรณีอื่น ๆ ที่คุณจะได้พบกับ undefined (เช่น ค่าที่ส่งกลับจาก Function ที่ไม่มีการ return) ซึ่งจะอธิบายในบทถัด ๆ ไป แต่จำง่าย ๆ ว่าค่านี้หมายถึงระบบยังไม่รู้จักค่าของมัน

ทำความเข้าใจ null

null คือค่าพิเศษที่เรากำหนดขึ้นมาเพื่อระบุว่าตัวแปรนี้ "ตั้งใจให้ว่างเปล่า" หรือ "ไม่มีข้อมูล":

let a = null
console.log(a) // null

การกำหนดค่าเป็น null คือการประกาศอย่างชัดเจนโดยผู้เขียนโปรแกรมว่าตัวแปรนี้ ไม่มีค่าอยู่ ณ ขณะนี้

แนวทางปฏิบัติที่ดีในการใช้งาน (Best Practices)

  1. ใช้ null เมื่อต้องการระบุว่าตัวแปร "ว่างเปล่า" โดยตั้งใจ

    let selectedFile = null
let currentTheme = null
let lastError = null

  2. ปล่อยให้ undefined เกิดขึ้นตามระบบของภาษา (ไม่ควรกำหนดเอง)

    let userName // จะถูกกำหนดค่าจริงในภายหลัง
let userPreferences // จะถูกโหลดข้อมูลจากเซิร์ฟเวอร์ในภายหลัง

  3. ใช้ null เมื่อต้องการล้างค่าข้อมูลเดิม

    let name = "Kyle"
// โค้ดส่วนอื่น ๆ เรียกใช้งาน name...
name = null // ล้างค่าออก เพื่อระบุว่าตอนนี้ name ไม่มีข้อมูลแล้ว

สรุปความแตกต่างที่สำคัญ

  • undefined - ตัวแปรนี้ยังไม่ได้ถูกกำหนดค่าใด ๆ (ระบบกำหนดให้โดยอัตโนมัติ)
  • null - ตัวแปรนี้ตั้งใจให้ไม่มีค่าหรือว่างเปล่า (ผู้เขียนโปรแกรมกำหนดเอง)

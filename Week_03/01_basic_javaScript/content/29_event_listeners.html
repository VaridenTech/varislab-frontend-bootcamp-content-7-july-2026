การใช้งาน Event Listener

Event Listener คือเครื่องมือสำคัญที่ช่วยทำให้หน้าเว็บของเราสามารถโต้ตอบกับผู้ใช้งานได้ (Interactive) โดยทำหน้าที่คอยตรวจจับและตอบสนองต่อพฤติกรรมต่าง ๆ เช่น การคลิกปุ่ม การพิมพ์ข้อความ การส่ง Form หรือการขยับเมาส์

1. Event คืออะไร?

Event (เหตุการณ์) คือสิ่งต่าง ๆ ที่เกิดขึ้นบนเว็บ Browser จากการกระทำของผู้ใช้หรือระบบ เช่น การคลิกปุ่ม (Click), การพิมพ์ในช่องข้อมูล (Input), การส่งข้อมูล Form (Submit) หรือการปรับขนาดหน้าจอ (Resize)

Event Listener (ตัวดักฟังเหตุการณ์) คือ Function พิเศษที่เราสั่งให้ไปเกาะติดอยู่กับองค์ประกอบหน้าเว็บ เพื่อคอยตรวจจับว่าเกิดเหตุการณ์นั้น ๆ ขึ้นหรือยัง และเมื่อเกิดเหตุการณ์ขึ้นแล้ว ระบบจะสั่งให้โค้ด JavaScript ที่เรากำหนดไว้ทำงานทันที

2. โครงสร้างไวยากรณ์พื้นฐาน

const button = document.querySelector("button")
button.addEventListener("click", () => {
  console.log("Button was clicked!")
})

Method addEventListener() จะรับ Argument หลัก 2 ตัวดังนี้:

  • eventType: ข้อความ (String) ระบุชนิดของเหตุการณ์ที่ต้องการตรวจจับ เช่น "click", "input", "submit"
  • function: Function คอลแบ็ก (Callback Function) ที่ต้องการให้ทำงานทันทีเมื่อเกิดเหตุการณ์นั้นขึ้น

3. ตัวอย่างการเพิ่ม Event Listener รูปแบบต่าง ๆ

const input = document.querySelector("input")
input.addEventListener("input", (e) => {
  console.log("Input value:", e.target.value)
})

document.addEventListener("click", () => console.log("Document was clicked"))

window.addEventListener("resize", () => {
  console.log("Window resized to:", window.innerWidth, window.innerHeight)
})

การผูก Event Listener หลายตัวเข้ากับองค์ประกอบเดียว

เราสามารถผูก Event Listener หลาย ๆ ตัวเข้ากับองค์ประกอบ HTML เดียวกันได้ โดยระบบจะไม่เขียนทับกัน แต่จะทำงานเรียงลำดับตามที่เราประกาศไว้:

button.addEventListener("click", () => console.log("First listener"))
button.addEventListener("click", () => console.log("Second listener"))

4. การยกเลิกและลบ Event Listener (removeEventListener)

หากเราต้องการลบ Event Listener ออกในภายหลัง เราจำเป็นต้อง ส่งอ้างอิงของ Function ตัวเดิม (Named Function Reference) เข้าไปใน Method removeEventListener():

function handleClick() { console.log("Clicked!") }
button.addEventListener("click", handleClick)
button.removeEventListener("click", handleClick)
// ❌ วิธีที่ผิด: การใช้ Arrow Function แบบนิรนาม (Anonymous) จะทำให้ลบไม่ได้ เนื่องจากระบบมองเป็น Function คนละตัวกันในหน่วยความจำ
button.addEventListener("click", () => console.log("Clicked!"))
button.removeEventListener("click", () => console.log("Clicked!"))

5. ทำความเข้าใจเกี่ยวกับ Event Object (e)

เมื่อเกิดเหตุการณ์ขึ้น Function คอลแบ็กจะได้รับ Argument ตัวหนึ่งโดยอัตโนมัติ เรียกว่า Event Object (นิยมตั้งชื่อตัวแปรย่อว่า e หรือ event) ซึ่งเก็บข้อมูลรายละเอียดทั้งหมดเกี่ยวกับเหตุการณ์ที่เกิดขึ้น ณ ขณะนั้น:

button.addEventListener("click", (event) => console.log(event))

คุณสมบัติ (Properties) ที่นิยมใช้งานของ Event Object

button.addEventListener("click", (e) => {
  console.log(e.target)
  console.log(e.currentTarget)
  console.log(e.clientX, e.clientY)
  console.log(e.ctrlKey, e.shiftKey, e.altKey)
})

ความแตกต่างระหว่าง target และ currentTarget

  • e.target: คือองค์ประกอบ HTML ตัวจริงเสียงจริงที่ผู้ใช้กระทำโดยตรง (ซึ่งอาจจะเป็น Tag ลูกที่อยู่ข้างในปุ่มอีกที เช่น รูปภาพหรือไอคอน)
  • e.currentTarget: คือองค์ประกอบ HTML ที่เราเอา Event Listener ไปผูกติดไว้จริง ๆ (เช่น ตัวปุ่มหลัก)

💡 คำแนะนำ: ในการทำงานส่วนใหญ่ ค่าที่เราต้องการนำมาใช้งานมักจะเป็น e.currentTarget เพราะมีความแน่นอนและไม่เปลี่ยนแปลงตามตำแหน่งที่ผู้ใช้คลิกโดน

6. ชนิดของ Event ที่นิยมใช้งานบ่อย ๆ

กลุ่มเหตุการณ์เมาส์ (Mouse Events):

  • click, dblclick, mouseenter, mouseleave, mouseover

กลุ่มเหตุการณ์โฟกัส (Focus Events):

  • focus, blur

กลุ่มเหตุการณ์ Input (Input Events):

  • input - ทำงานทันทีในทุก ๆ ครั้งที่มีการพิมพ์หรือค่าเปลี่ยน
  • change - ทำงานเมื่อผู้ใช้พิมพ์เสร็จและกดคลิกออกนอกช่องกรอกข้อมูล (เสีย Focus)

กลุ่มเหตุการณ์หน้าจอ (Window Events):

  • resize - เมื่อขนาดหน้าต่างเปลี่ยน

7. การยกเลิกพฤติกรรมเริ่มต้นของ Browser ด้วย preventDefault()

องค์ประกอบ HTML บางตัวจะมีพฤติกรรมเริ่มต้นที่ Browser กำหนดมาให้โดยอัตโนมัติ เช่น เมื่อกดปุ่มส่ง Form หน้าเว็บจะรีโหลดใหม่ หรือเมื่อคลิก Link หน้าเว็บจะเปลี่ยนเส้นทางไปหน้าอื่น เราสามารถใช้ Method e.preventDefault() เพื่อสั่งระงับพฤติกรรมเริ่มต้นเหล่านี้ชั่วคราว เพื่อให้เราสามารถประมวลผลข้อมูลด้วย JavaScript เองก่อนได้:

const form = document.querySelector("form")
form.addEventListener("submit", (e) => {
  e.preventDefault()
  const formData = new FormData(e.currentTarget)
  console.log("Username:", formData.get("username"))
})
link.addEventListener("click", (e) => e.preventDefault())
document.addEventListener("contextmenu", (e) => e.preventDefault())

8. ข้อผิดพลาดที่พบบ่อย (Common Mistakes)

  1. การใส่เครื่องหมายวงเล็บเรียกใช้งาน Function ทันที

    // ❌ วิธีที่ผิด: การใส่ () จะทำให้ Function ทำงานทันทีตั้งแต่หน้าเว็บโหลดเสร็จ ไม่ใช่ทำงานเมื่อเกิดการคลิก
button.addEventListener("click", handleClick())

// ✅ วิธีที่ถูกต้อง: ส่งเฉพาะชื่อ Function เพื่อบอกระบบว่าให้เรียกใช้เมื่อมีการคลิกเท่านั้น
button.addEventListener("click", handleClick)

  2. การสับสนระหว่าง target และ currentTarget

    button.addEventListener("click", (e) => {
  console.log(e.target) // ⚠️ อาจเป็น Tag ลูกข้างในปุ่ม
  console.log(e.currentTarget) // ✅ ปลอดภัยและเป็นตัวปุ่มหลักเสมอ
})

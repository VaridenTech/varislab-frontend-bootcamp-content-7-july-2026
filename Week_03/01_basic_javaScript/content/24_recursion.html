การใช้งาน Function เรียกตัวเอง (Recursion)

Recursion (การเรียกตัวเองซ้ำ) คือเทคนิคการเขียนโปรแกรมที่ Function หนึ่ง ๆ จะทำการ เรียกใช้งานตัวเอง (Self-calling) เพื่อแก้ปัญหาที่มีลักษณะซับซ้อน โดยการย่อยปัญหาใหญ่นั้นออกเป็นปัญหาย่อย ๆ ที่มีรูปแบบโครงสร้างเหมือนเดิมแต่มีขนาดเล็กลงเรื่อย ๆ

💡 คำแนะนำ: หัวข้อนี้เป็นหนึ่งในแนวคิดที่เข้าใจยากที่สุดสำหรับผู้เริ่มต้นเขียนโปรแกรม แนะนำให้ค่อย ๆ ทำความเข้าใจกลไกการทำงานเบื้องหลังและลองวาดภาพตามไปพร้อม ๆ กันนะครับ

1. ตัวอย่างการทำงานของ Recursion เบื้องต้น

function printNumbers(number) {
  if (number > 10) return // Base Case: เงื่อนไขสิ้นสุดเพื่อหยุดการเรียกตัวเองซ้ำ
  console.log(number)
  printNumbers(number + 1) // Recursive Case: Function เรียกตัวเองพร้อมส่งค่าใหม่เข้าไป
}
printNumbers(1) // แสดงตัวเลข 1 ถึง 10

ซึ่งโค้ด Recursion ด้านบนสามารถเขียนทดแทนด้วยคำสั่ง For Loop ทั่วไปได้ดังนี้:

for (let i = 1; i <= 10; i++) console.log(i)

2. สองส่วนประกอบสำคัญของ Function Recursion

1. เงื่อนไขสิ้นสุด (Base Case)

คือเงื่อนไขสำคัญที่บอกให้ Function หยุดเรียกตัวเองและส่งค่ากลับออกไปทันที เพื่อป้องกันไม่ให้ Function ทำงานวนซ้ำไปเรื่อย ๆ อย่างไม่มีที่สิ้นสุด:

if (number > 10) return

⚠️ ข้อควรระวัง: หากเราเขียน Function Recursion โดยไม่มี Base Case หรือกำหนดเงื่อนไขผิดพลาด จะทำให้เกิดการเรียกตัวเองซ้ำไม่สิ้นสุดจนหน่วยความจำเต็ม และระบบจะแจ้งข้อผิดพลาด Stack Overflow (Maximum call stack size exceeded) ส่งผลให้โปรแกรมค้างหรือล่มทันที:

// ❌ ตัวอย่างโค้ดที่ไม่มี Base Case ทำให้โปรแกรมล่ม!
function printHi() {
  console.log("Hi")
  printHi() // RangeError: Maximum call stack size exceeded
}
printHi()

2. การเรียกตัวเองซ้ำ (Recursive Case)

คือส่วนของโค้ดที่ Function จะเรียกใช้งานตัวเองอีกครั้ง แต่ต้องมั่นใจว่า ค่า Argument ที่ส่งเข้าไปใหม่นั้นจะต้องขยับเข้าใกล้ Base Case มากขึ้นเรื่อย ๆ ในทุก ๆ รอบ:

printNumbers(number + 1) // เรียกตัวเองซ้ำโดยส่งค่าตัวเลขที่เพิ่มขึ้นทีละ 1 เข้าไป

3. ทำความเข้าใจกลไก Call Stack กับ Recursion

เมื่อ Function ทำการเรียกตัวเองซ้ำ JavaScript Engine จะนำ Function แต่ละรอบไปซ้อนทับกันในหน่วยความจำที่เรียกว่า Call Stack (เหมือนกับการวางจานซ้อนกันขึ้นไปเรื่อย ๆ) โดย Function ที่อยู่บนสุดจะต้องทำงานให้เสร็จและถูกหยิบออกไปก่อน Function ด้านล่างจึงจะทำงานต่อได้:

function countdown(n) {
  console.log(n)
  if (n <= 0) return
  console.log("Before")
  countdown(n - 1)
  console.log("After")
}
countdown(3)

ลำดับการทำงานใน Call Stack (Execution Order):

countdown(3) ทำงาน -> แสดง 3 -> เรียก countdown(2)
  countdown(2) ทำงาน -> แสดง 2 -> เรียก countdown(1)
    countdown(1) ทำงาน -> แสดง 1 -> เรียก countdown(0)
      countdown(0) ทำงาน -> เข้าเงื่อนไข Base Case -> สิ้นสุดและส่งค่ากลับ (Pop ออกจาก Stack)
    countdown(1) ทำงานส่วนที่เหลือต่อ -> แสดง "After" -> ส่งค่ากลับ (Pop ออก)
  countdown(2) ทำงานส่วนที่เหลือต่อ -> แสดง "After" -> ส่งค่ากลับ (Pop ออก)
countdown(3) ทำงานส่วนที่เหลือต่อ -> แสดง "After" -> สิ้นสุดการทำงานทั้งหมด

4. ตัวอย่างการใช้งานจริง: การจัดการโครงสร้างข้อมูลแบบต้นไม้ (Tree Structures)

ประโยชน์สูงสุดของ Recursion คือการนำมาจัดการกับโครงสร้างข้อมูลที่มีความซับซ้อนและซ้อนกันหลายชั้น เช่น โครงสร้างโฟลเดอร์และไฟล์ (File System) หรือโครงสร้างข้อมูลแบบต้นไม้ (Tree) ซึ่งยากต่อการจัดการด้วยการวนลูปทั่วไป:

// ตัวอย่างโครงสร้างระบบโฟลเดอร์และไฟล์
const fileSystem = {
  name: "root", type: "folder",
  children: [
    { name: "documents", type: "folder", children: [
      { name: "resume.pdf", type: "file" },
      { name: "notes.txt", type: "file" },
    ]},
    { name: "photos", type: "folder", children: [
      { name: "vacation.jpg", type: "file" },
    ]},
    { name: "readme.txt", type: "file" },
  ],
}

function countFiles(item) {
  if (item.type === "file") return 1 // Base case
  let count = 0
  for (const child of item.children) {
    count += countFiles(child) // เรียกใช้งาน Function countFiles ซ้ำสำหรับโฟลเดอร์ย่อย
  }
  return count
}
console.log(countFiles(fileSystem)) // ผลลัพธ์: 4

5. ข้อผิดพลาดที่พบบ่อยใน Recursion (Common Pitfalls)

  1. การลืมเขียนหรือกำหนด Base Case ผิดพลาด

    // ✅ ตัวอย่างที่ถูกต้อง: มี Base Case ป้องกันลูปไม่สิ้นสุด
function countdown(n) {
  if (n <= 0) return
  console.log(n)
  countdown(n - 1)
}

// ❌ ตัวอย่างที่ผิด: ไม่มี Base Case ทำให้เกิด Stack Overflow
function badCountdown(n) {
  console.log(n)
  badCountdown(n - 1)
}

  2. ค่า Argument ไม่ขยับเข้าใกล้ Base Case

    // ✅ ตัวอย่างที่ถูกต้อง: ค่า n ลดลงเรื่อย ๆ จนถึงเงื่อนไขหยุด
function countdown(n) {
  if (n <= 0) return
  console.log(n)
  countdown(n - 1)
}

// ❌ ตัวอย่างที่ผิด: ค่า n ไม่เปลี่ยนแปลง ทำให้ Function เรียกตัวเองซ้ำไปเรื่อย ๆ
function badCountdown(n) {
  if (n <= 0) return
  console.log(n)
  countdown(n)
}

6. สรุปแนวทางการเลือกใช้งาน Recursion

✅ ควรเลือกใช้ Recursion เมื่อ:

  • ต้องทำงานกับโครงสร้างข้อมูลแบบต้นไม้ (Tree), กราฟ (Graph) หรือข้อมูลที่มีการซ้อนกันลึกหลายชั้น
  • ปัญหานั้นสามารถแบ่งออกเป็นปัญหาย่อย ๆ ที่มีขั้นตอนการทำงานเหมือนเดิมได้ชัดเจน
  • การเขียนแบบ Recursion ช่วยให้โค้ดมีความสั้น กระชับ และอ่านเข้าใจง่ายกว่าการเขียนลูปซ้อนกันหลาย ๆ ชั้น
  • ต้องการประมวลผลข้อมูลที่มีความสัมพันธ์เป็นลำดับขั้น (Hierarchical Data)

❌ ควรหลีกเลี่ยงและใช้ Loop ทั่วไปเมื่อ:

  • การวนลูปแบบธรรมดา (For/While) สามารถแก้ปัญหาได้ดีและเข้าใจง่ายอยู่แล้ว
  • ต้องการประสิทธิภาพการทำงานที่รวดเร็วและใช้หน่วยความจำน้อยที่สุด (เนื่องจาก Recursion จะใช้พื้นที่บน Call Stack ค่อนข้างมาก)
  • ตรรกะการเรียกตัวเองซ้ำมีความซับซ้อนเกินไปจนทำให้ทีมพัฒนาเข้าใจยาก

ลองทำแบบฝึกหัด

จงเขียน Function Recursive เพื่อค้นหาค่าตัวเลขที่สูงที่สุด (Maximum Value) จาก Array ที่มีการซ้อนกันหลายชั้น (Nested Array) ดังนี้:

const data = [1, [2, 3], [4, [5, 6]], 7]
// ผลลัพธ์ที่ควรได้คือ: 7
ดูเฉลย 
function findMax(arr) {
  let max = -Infinity
  for (const item of arr) {
    if (Array.isArray(item)) {
      const subMax = findMax(item)
      max = Math.max(max, subMax)
    } else {
      max = Math.max(max, item)
    }
  }
  return max
}
const data = [1, [2, 3], [4, [5, 6]], 7]
console.log(findMax(data)) // 7

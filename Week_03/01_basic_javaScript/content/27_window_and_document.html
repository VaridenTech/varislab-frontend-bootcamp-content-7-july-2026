Window และ Document Object

เมื่อ JavaScript ทำงานในเบราว์เซอร์ จะเข้าถึง Global object พื้นฐานสองตัวคือ window และ document ซึ่งเป็นตัวเชื่อมระหว่างโค้ด JavaScript กับหน้าเว็บ

Window Object

window แทนหน้าต่างเบราว์เซอร์และเป็น Global object ของ JavaScript ทั้งหมดที่ทำงานในหน้าต่างนั้น ทุกสิ่งที่เข้าถึงแบบ Global แท้จริงคือพร็อพเพอร์ตีของ Window

ลักษณะ Global ของ Window

// ทำงานเทียบเท่ากัน
console.log("Hello")
window.console.log("Hello")
alert("Hi there!")
window.alert("Hi there!")

เมื่อใช้ฟังก์ชันหรือตัวแปร Global JavaScript จะถือว่าคุณหมายถึง window.something

สำรวจ Window Object

console.log(window)

ใน Console จะเห็นว่ามีพร็อพเพอร์ตีและเมธอดจำนวนมาก แต่ไม่จำเป็นต้องจำทั้งหมด

เมื่อใดต้องใช้ window

  1. ชื่อขัดแย้งกัน: ใช้ window เพื่อเข้าถึงเวอร์ชัน Global

    const alert = "This is my custom alert message"
// alert("Hello") // ❌ Error
window.alert(alert) // ✅ ใช้ฟังก์ชัน alert ของ Window

  2. สร้างตัวแปร Global:

    window.myGlobalVar = "Hello, World!"
console.log(myGlobalVar) // ✅ "Hello, World!"

    โดยทั่วไปไม่แนะนำ เพราะทำให้โค้ดดูแลรักษายากขึ้น

Document Object

document แทนเอกสาร HTML ที่โหลดในหน้าต่าง HTML ของหน้ามักเรียกว่า DOM (Document Object Model)

Document Object คืออะไร?

console.log(document)

จะเห็นโครงสร้าง HTML ทั้งหมด ตั้งแต่ <!DOCTYPE html> จนถึงแท็กปิด </html> และ Document เป็นพร็อพเพอร์ตีของ Window:

console.log(document)
console.log(window.document)

พร็อพเพอร์ตีสำคัญของ Document

console.log(document.documentElement) // Element <html>
console.log(document.body) // Element <body>
console.log(document.head) // Element <head>
console.log(document.URL) // URL ปัจจุบัน

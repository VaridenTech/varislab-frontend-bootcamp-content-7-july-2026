การแปลงชนิดข้อมูล

การแปลงชนิดข้อมูลคือวิธีที่ JavaScript แปลงข้อมูลชนิดหนึ่งเป็นอีกชนิดหนึ่งโดยอัตโนมัติ แม้จะสะดวก แต่ก็มักนำไปสู่พฤติกรรมที่ชวนสับสนและไม่คาดคิด

ทบทวนชนิดข้อมูลใน JavaScript

ก่อนเจาะลึกเรื่องการแปลงชนิดข้อมูล มาทบทวนชนิดข้อมูลหลักใน JavaScript กัน:

  • Number - 1, 2.5, -10
  • String - "hello", "123"
  • Boolean - true, false
  • null - แสดงถึงการไม่มีค่าโดยตั้งใจ
  • undefined - แสดงถึงค่าที่ยังไม่ได้กำหนดหรือขาดหายไป

การแปลงชนิดข้อมูลแบบชัดเจน

การแปลงชนิดข้อมูลแบบชัดเจนคือเมื่อ คุณ สั่งให้ JavaScript แปลงชนิดหนึ่งเป็นอีกชนิดหนึ่ง

การแปลง String เป็น Number

const a = "1"
console.log(typeof a) // "string"

// แปลงเป็นจำนวนเต็ม
const numberA = parseInt(a)
console.log(typeof numberA) // "number"

parseInt กับ parseFloat

  • parseInt - แปลงเป็นจำนวนเต็ม
  • parseFloat - แปลงเป็นเลขทศนิยม
let decimal = "1.3"

console.log(parseInt(decimal)) // 1 (ตัดส่วนทศนิยมออก)
console.log(parseFloat(decimal)) // 1.3 (คงทศนิยมไว้)

การแปลงตัวแปรเป็น String

const num = 1.34
const stringNum = num.toString()

console.log(typeof stringNum) // "string"

การแปลงชนิดข้อมูลโดยปริยาย

การแปลงชนิดข้อมูลโดยปริยายเกิดขึ้นเมื่อ JavaScript แปลงชนิดให้คุณ โดยอัตโนมัติ โดยที่คุณไม่ได้สั่ง นี่คือจุดที่อาจทำให้สับสน

ปัญหาของตัวดำเนินการบวก +

เมื่อใช้ + กับ String และ Number JavaScript จะแปลงตัวเลขเป็น String:

const a = 1 // number
const b = "3" // string

console.log(a + b) // "13" (ไม่ใช่ 4)

เหมือนกับการเขียนว่า:

console.log(a.toString() + b) // "13"

แก้ไขได้ด้วยการแปลงชนิดข้อมูลแบบชัดเจน:

console.log(a + parseInt(b)) // 4 (การบวกที่ถูกต้อง)

ตัวดำเนินการอื่นทำงานต่างออกไป

ต่างจาก + ตรงที่ตัวดำเนินการทางคณิตศาสตร์อื่นจะแปลง String เป็น Number:

const a = 3 // number
const b = "1" // string

console.log(a - b) // 2 (แปลง "1" เป็น 1 แล้วคำนวณ 3 - 1)
console.log(a * b) // 3 (แปลง "1" เป็น 1 แล้วคำนวณ 3 * 1)

ควรใช้แบบใด?

โดยทั่วไปควรหลีกเลี่ยงการแปลงโดยปริยาย เพราะอาจทำให้ได้ผลลัพธ์ที่ไม่คาดคิด ให้ใช้ฟังก์ชันแปลงแบบชัดเจน เช่น parseInt, parseFloat หรือ toString เมื่อต้องการแปลงชนิดข้อมูล

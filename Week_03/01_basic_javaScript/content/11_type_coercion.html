การแปลงชนิดข้อมูล (Type Coercion)

การแปลงชนิดข้อมูล (Type Coercion) คือพฤติกรรมที่ JavaScript ทำการแปลงข้อมูลจากชนิดหนึ่งไปเป็นอีกชนิดหนึ่งโดยอัตโนมัติในขณะประมวลผล แม้ว่าบางครั้งจะช่วยอำนวยความสะดวก แต่บ่อยครั้งก็มักจะนำไปสู่ผลลัพธ์ที่แปลกประหลาด ชวนสับสน และไม่คาดคิดได้เช่นกัน

ทบทวนชนิดข้อมูลพื้นฐาน (Primitive Types)

ก่อนที่เราจะไปดูเรื่องการแปลงชนิดข้อมูล มาทบทวนชนิดข้อมูลหลัก ๆ ใน JavaScript กันก่อน:

  • Number - 1, 2.5, -10
  • String - "hello", "123"
  • Boolean - true, false
  • null - ใช้ระบุว่าตั้งใจให้ไม่มีค่าหรือว่างเปล่า
  • undefined - ใช้ระบุว่าตัวแปรยังไม่ได้ถูกกำหนดค่าใด ๆ

1. การแปลงชนิดข้อมูลแบบชัดเจน (Explicit Coercion)

Explicit Coercion คือการที่เรา (ผู้เขียนโปรแกรม) เป็นคนสั่งการอย่างชัดเจนให้ JavaScript แปลงข้อมูลจากชนิดหนึ่งไปเป็นอีกชนิดหนึ่งผ่าน Function ต่าง ๆ

การแปลงข้อความ (String) เป็นตัวเลข (Number)

const a = "1"
console.log(typeof a) // "string"

// แปลงเป็นจำนวนเต็ม (Integer)
const numberA = parseInt(a)
console.log(typeof numberA) // "number"

ความแตกต่างระหว่าง parseInt และ parseFloat

  • parseInt - parseInt() - ใช้แปลงข้อความเป็นจำนวนเต็ม (ไม่มีทศนิยม)
  • parseFloat - parseFloat() - ใช้แปลงข้อความเป็นเลขทศนิยม
let decimal = "1.3"

console.log(parseInt(decimal)) // 1 (ตัดทศนิยมทิ้ง)
console.log(parseFloat(decimal)) // 1.3 (คงทศนิยมไว้)

การแปลงข้อมูลอื่น ๆ เป็นข้อความ (String)

const num = 1.34
const stringNum = num.toString()

console.log(typeof stringNum) // "string"

2. การแปลงชนิดข้อมูลโดยปริยาย (Implicit Coercion)

Implicit Coercion คือการที่ JavaScript Engine ทำการแปลงชนิดข้อมูลให้เรา โดยอัตโนมัติ โดยที่เราไม่ได้เขียนสั่งการ ซึ่งมักจะเกิดขึ้นเมื่อเรานำข้อมูลต่างชนิดกันมาทำงานร่วมกัน และนี่คือจุดที่สร้างความสับสนให้กับนักพัฒนามากที่สุด

ปัญหาของตัวดำเนินการบวก (+)

เมื่อเราใช้เครื่องหมาย + ระหว่างข้อมูลประเภท String และ Number ตัว JavaScript จะเลือกแปลง Number ให้กลายเป็น String แล้วนำมาต่อกันแทนการบวกเลข:

const a = 1 // number
const b = "3" // string

console.log(a + b) // "13" (ไม่ใช่ 4 เพราะเป็นการต่อข้อความ)

ซึ่งเบื้องหลังการทำงานจะเทียบเท่ากับการเขียนแบบนี้:

console.log(a.toString() + b) // "13"

วิธีแก้ไขคือ เราต้องแปลงชนิดข้อมูลให้เป็นตัวเลขอย่างชัดเจนก่อนนำมาบวกกัน:

console.log(a + parseInt(b)) // 4 (ผลลัพธ์เป็น 4 บวกเลขอย่างถูกต้อง)

ตัวดำเนินการทางคณิตศาสตร์อื่น ๆ (-, *, /)

ต่างจากเครื่องหมายบวกอย่างสิ้นเชิง! ตัวดำเนินการทางคณิตศาสตร์อื่น ๆ เช่น ลบ (-), คูณ (*), และ หาร (/) จะแปลงข้อมูล String ให้กลายเป็น Number โดยอัตโนมัติเพื่อทำการคำนวณ:

const a = 3 // number
const b = "1" // string

console.log(a - b) // 2 (แปลง "1" เป็นตัวเลข 1 แล้วคำนวณ 3 - 1)
console.log(a * b) // 3 (แปลง "1" เป็นตัวเลข 1 แล้วคำนวณ 3 * 1)

สรุปแนวทางปฏิบัติที่ดี (Best Practices)

In ทางปฏิบัติ แนะนำให้หลีกเลี่ยงการปล่อยให้ JavaScript แปลงชนิดข้อมูลโดยปริยาย (Implicit Coercion) เสมอ เพราะอาจทำให้เกิด Bug ที่หาสาเหตุได้ยาก ให้เปลี่ยนมาใช้การแปลงชนิดข้อมูลแบบชัดเจน (Explicit Coercion) เช่น การเรียกใช้ parseInt(), parseFloat() หรือ toString() ทุกครั้งเมื่อต้องการเปลี่ยนประเภทข้อมูล เพื่อให้โค้ดของเราทำงานได้อย่างถูกต้อง แม่นยำ และเข้าใจง่ายสำหรับทุกคน

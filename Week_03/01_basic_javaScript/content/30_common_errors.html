ข้อผิดพลาดที่พบบ่อยใน JavaScript (Common Errors)

ไม่ว่าคุณจะเป็นนักพัฒนามือใหม่หรือผู้เชี่ยวชาญ การพบเจอข้อผิดพลาด (Errors) ในการเขียนโค้ดเป็นเรื่องที่หลีกเลี่ยงไม่ได้ การทำความเข้าใจประเภทของข้อผิดพลาดและรู้วิธีการอ่านข้อความแจ้งเตือน (Error Messages) จะช่วยให้คุณสามารถค้นหาและแก้ไข Bug (Debug) ได้อย่างรวดเร็วและมีประสิทธิภาพมากขึ้น

1. ประเภทของข้อผิดพลาดในโปรแกรม

  1. Syntax Error (ข้อผิดพลาดทางไวยากรณ์): เกิดจากการเขียนโค้ดผิดกฎไวยากรณ์ของภาษา ทำให้โปรแกรมไม่สามารถคอมไพล์หรือรันได้ตั้งแต่แรก
  2. Runtime Error (ข้อผิดพลาดขณะทำงาน): โค้ดผ่านไวยากรณ์รันได้ปกติ แต่เกิดปัญหาขึ้นในขณะที่โปรแกรมกำลังทำงาน ส่งผลให้โปรแกรมหยุดทำงานกะทันหัน
  3. Logical Error (ข้อผิดพลาดทางตรรกะ): โปรแกรมรันได้ผ่านฉลุยไม่มี Error แจ้งเตือนใด ๆ แต่ผลลัพธ์ที่ได้กลับไม่ถูกต้องตามที่ต้องการ (มักเรียกว่า Bug)

ตัวอย่าง Syntax Error (ข้อผิดพลาดทางไวยากรณ์)

มักเกิดจากการลืมพิมพ์เครื่องหมายสำคัญ เช่น เครื่องหมายจุลภาค (Comma ,) หรือวงเล็บปิดไม่ครบ:

// ❌ ผิด: ลืมใส่เครื่องหมายจุลภาคคั่นระหว่าง Property
const person = { name: "John" age: 30 }

// ✅ แก้ไขถูกต้อง
const person = { name: "John", age: 30 }
// ❌ ผิด: ลืมใส่วงเล็บปิดเงื่อนไขของ if
if (age > 18 { console.log("Adult") }

// ✅ แก้ไขถูกต้อง
if (age > 18) { console.log("Adult") }

ตัวอย่าง Runtime Error (ข้อผิดพลาดขณะทำงาน)

  1. 1. การพิมพ์ชื่อตัวแปรสะกดผิด (ReferenceError)

    const userName = "john123"
console.log(username) // ❌ ReferenceError เพราะสะกดด้วยตัวพิมพ์เล็กทั้งหมด
console.log(userName) // ✅ สะกดถูกต้องตามที่ประกาศไว้

  2. 2. การเข้าถึงคุณสมบัติของค่าที่เป็น null หรือ undefined (TypeError)

    const user = getUser()
// console.log(user.name) // ❌ หาก getUser() ส่งคืนค่า null โปรแกรมจะแครชทันที
if (user) console.log(user.name) // ✅ ตรวจสอบความปลอดภัยก่อนใช้งาน

  3. 3. การพยายามกำหนดค่าใหม่ให้กับตัวแปรประเภทค่าคงที่ (TypeError)

    const age = 30
// age = 31 // ❌ TypeError: Assignment to constant variable.

let newAge = 30
newAge = 31 // ✅ สามารถแก้ไขค่าได้ปกติ

ตัวอย่าง Logical Error (ข้อผิดพลาดทางตรรกะ)

  1. 1. ข้อผิดพลาดคลาดเคลื่อนหนึ่งตำแหน่ง (Off-by-One Error)

    const items = ["apple", "banana", "orange"]
for (let i = 0; i <= items.length; i++) console.log(items[i]) // ❌ รอบสุดท้ายจะดึงค่าตำแหน่งที่ 3 ซึ่งไม่มีอยู่จริง (undefined)
for (let i = 0; i < items.length; i++) console.log(items[i]) // ✅ ดึงข้อมูลได้ถูกต้องครบถ้วน

  2. 2. การสับสนระหว่างเครื่องหมายกำหนดค่าและเครื่องหมายเปรียบเทียบ

    // ❌ ผิด: ใช้เครื่องหมายเท่ากับตัวเดียว = (เป็นการกำหนดค่า) แทนที่จะเป็น === (การเปรียบเทียบ)
if (name = "Sally") console.log("Is Sally")

// ✅ เปรียบเทียบความเท่ากันอย่างถูกต้อง
if (name === "Sally") console.log("Is Sally")

2. ข้อผิดพลาดที่พบบ่อยเกี่ยวกับการจัดการหน้าเว็บ (DOM Errors)

  1. 1. การพยายามใช้งาน Element ที่ไม่มีอยู่จริง (Null Element)

    const button = document.querySelector(".submit-btn")
// button.addEventListener("click", handleClick) // ❌ หากไม่มีปุ่ม Class นี้บนหน้าเว็บ โปรแกรมจะแครชทันที
if (button) button.addEventListener("click", handleClick) // ✅ ตรวจสอบก่อนว่ามีองค์ประกอบอยู่จริง
else console.warn("Submit button not found")

  2. 2. ไฟล์ Script ถูกโหลดและรันก่อนที่หน้าเว็บ HTML จะสร้างเสร็จ

    // ในไฟล์ HTML ควรใส่ Attribute defer เพื่อให้ Browser โหลด HTML ให้เสร็จก่อนรัน Script
<script src="script.js" defer></script>

3. วิธีการอ่านข้อความแจ้งเตือนข้อผิดพลาด (Error Messages)

TypeError: Cannot read property 'name' of null
    at getUserName (script.js:15:20)
    at main (script.js:8:5)
  • ประเภทของ Error (Type): บอกประเภทของข้อผิดพลาด เช่น TypeError
  • ข้อความอธิบาย (Message): รายละเอียดที่บอกว่าเกิดอะไรขึ้น (เช่น ไม่สามารถอ่านคุณสมบัติ 'name' ของค่าที่เป็น null ได้)
  • ประวัติการเรียก Function (Stack Trace): ลำดับขั้นตอนการทำงานของ Function ที่นำไปสู่การเกิด Error พร้อมระบุชื่อไฟล์และเลขบรรทัดอย่างละเอียด (เช่น บรรทัดที่ 15 ในไฟล์ script.js)

ข้อความแจ้งเตือนที่พบบ่อยและคำอธิบาย

// ReferenceError: variable is not defined → มีการเรียกใช้งานตัวแปรที่ยังไม่ได้ประกาศ
// TypeError: Cannot read property 'X' of null/undefined → พยายามเข้าถึงคุณสมบัติ X บนตัวแปรที่เป็น null หรือ undefined
// TypeError: X is not a function → พยายามเรียกใช้งานตัวแปร X ในฐานะ Function ทั้งที่มันไม่ใช่ Function
// SyntaxError: Unexpected token → มีการพิมพ์สะกดผิดหรือเครื่องหมายบางอย่างขาดหายไป
// RangeError: Maximum call stack size exceeded → เกิดการเรียกตัวเองซ้ำของ Function (Recursion) แบบไม่สิ้นสุดจนหน่วยความจำเต็ม

4. เคล็ดลับการค้นหาและแก้ไขข้อผิดพลาด (Debugging Tips)

  1. ตั้งสติและอ่านข้อความ Error ให้ครบถ้วน โดยเฉพาะการดูเลขบรรทัดที่เกิดปัญหา
  2. ตรวจสอบการสะกดคำ ตัวพิมพ์เล็ก-ใหญ่ และเครื่องหมายต่าง ๆ ให้ละเอียด
  3. ใช้คำสั่ง console.log() เพื่อตรวจสอบค่าของตัวแปรในแต่ละขั้นตอนการทำงาน
  4. ลองปิด Comment โค้ดบางส่วนชั่วคราวเพื่อแยกแยะและจำกัดวงของปัญหา
  5. เปิดแท็บ Console ของ Browser (DevTools) ทิ้งไว้เสมอในขณะพัฒนาหน้าเว็บ

ข้อผิดพลาด JavaScript ที่พบบ่อย

นักพัฒนา JavaScript ทุกคนพบข้อผิดพลาด การเข้าใจประเภทหลักและวิธีระบุปัญหาจะช่วยให้ทำงานได้มีประสิทธิภาพขึ้น

ประเภทข้อผิดพลาด

  1. Syntax Error: ไวยากรณ์ผิดจนโค้ดรันไม่ได้
  2. Runtime Error: ปัญหาเกิดระหว่างทำงานและทำให้โค้ดหยุด
  3. Logical Error (Bug): โค้ดรันได้แต่ผลไม่ถูกต้อง

Syntax Error

เกิดเมื่อโค้ดไม่เป็นไปตามกฎไวยากรณ์ เช่น ไม่มีจุลภาค หรือวงเล็บไม่ตรงกัน:

// ❌ ไม่มีจุลภาค
const person = { name: "John" age: 30 }
// ✅ แก้ไขแล้ว
const person = { name: "John", age: 30 }
// ❌ ไม่มีวงเล็บปิด
if (age > 18 { console.log("Adult") }
// ✅ แก้ไขแล้ว
if (age > 18) { console.log("Adult") }

Runtime Error

  1. พิมพ์ชื่อตัวแปรผิด

    const userName = "john123"
console.log(username) // ❌ ReferenceError
console.log(userName) // ✅

  2. เข้าถึงพร็อพเพอร์ตีของ Undefined หรือ Null

    const user = getUser()
// console.log(user.name) // ❌
if (user) console.log(user.name) // ✅

  3. กำหนดค่าใหม่ให้ Const

    const age = 30
// age = 31 // ❌
let newAge = 30
newAge = 31 // ✅

Logical Error

  1. คลาดหนึ่งตำแหน่ง

    const items = ["apple", "banana", "orange"]
for (let i = 0; i <= items.length; i++) console.log(items[i]) // ❌
for (let i = 0; i < items.length; i++) console.log(items[i]) // ✅

  2. การกำหนดค่ากับการเปรียบเทียบ

    // ❌ ใช้ = แทน ===
if (name = "Sally") console.log("Is Sally")
// ✅
if (name === "Sally") console.log("Is Sally")

ข้อผิดพลาดเกี่ยวกับ DOM

  1. ไม่พบ Element: ตรวจว่ามี Element ก่อนใช้งาน

    const button = document.querySelector(".submit-btn")
if (button) button.addEventListener("click", handleClick)
else console.warn("Submit button not found")

  2. Script โหลดก่อน DOM: ใช้แอตทริบิวต์ defer

    <script src="script.js" defer></script>

วิธีอ่านข้อความ Error

TypeError: Cannot read property 'name' of null
    at getUserName (script.js:15:20)
    at main (script.js:8:5)
  • ประเภท: TypeError
  • ข้อความ: อธิบายสิ่งที่ผิด
  • Stack trace: แสดงตำแหน่งและลำดับการเรียก

ข้อความที่พบบ่อย

// ReferenceError: variable is not defined → ใช้ตัวแปรที่ยังไม่ประกาศ
// TypeError: Cannot read property 'X' → เข้าถึงพร็อพเพอร์ตีบน undefined/null
// TypeError: X is not a function → เรียกสิ่งที่ไม่ใช่ฟังก์ชัน
// SyntaxError: Unexpected token → พิมพ์ผิดหรือเครื่องหมายหาย
// RangeError: Maximum call stack size exceeded → Recursion ไม่สิ้นสุด

เคล็ดลับ Debug อย่างรวดเร็ว

  1. อ่านข้อความ Error ทั้งหมดและดูเลขบรรทัด
  2. ตรวจคำสะกดและเครื่องหมาย
  3. ใช้ console.log อย่างมีกลยุทธ์
  4. Comment โค้ดบางส่วนเพื่อแยกปัญหา
  5. เปิด Console ของเบราว์เซอร์ไว้เสมอ

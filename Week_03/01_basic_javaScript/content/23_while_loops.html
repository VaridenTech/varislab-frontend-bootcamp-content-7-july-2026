การใช้งาน While Loop

While Loop คือคำสั่งควบคุมการทำงานซ้ำ (Loop) ที่เหมาะสำหรับกรณีที่เรา ไม่ทราบจำนวนรอบการทำงานที่แน่นอนล่วงหน้า โดยโปรแกรมจะทำงานซ้ำไปเรื่อย ๆ ตราบใดที่เงื่อนไขที่กำหนดไว้ยังคงเป็นจริง (True)

1. โครงสร้างไวยากรณ์พื้นฐานของ While Loop

คำสั่ง While Loop จะเริ่มต้นด้วย Keyword while ตามด้วยเงื่อนไขที่ต้องการตรวจสอบภายในวงเล็บ () และบล็อกคำสั่งที่ต้องการให้ทำงานซ้ำภายในวงเล็บปีกกา {}:

let i = 0
while (i < 5) {
  // ชุดคำสั่งที่ต้องการให้ทำงานซ้ำ
  console.log(`Count: ${i}`)
  i++ // ⚠️ สำคัญมาก: ต้องอัปเดตค่าตัวแปรควบคุมเพื่อป้องกันการเกิด Infinite Loop!
}

ลูปจะทำงานซ้ำไปเรื่อย ๆ ตราบใดที่ผลลัพธ์ของเงื่อนไขในวงเล็บยังคงเป็น true และจะหยุดทำงานทันทีเมื่อเงื่อนไขกลายเป็น false

2. การเปรียบเทียบการแปลง For Loop เป็น While Loop

เราสามารถเขียนคำสั่ง For Loop และ While Loop ให้ทำงานทดแทนกันได้ โดยมีโครงสร้างเปรียบเทียบดังนี้:

// For loop
for (let i = 0; i < 5; i++) {
  console.log(i)
}

// การเขียนแบบ While Loop ที่ให้ผลลัพธ์เหมือนกัน
let i = 0 // 1. กำหนดค่าเริ่มต้นของตัวแปรควบคุม (เขียนไว้นอก Loop)
while (i < 5) { // 2. ตรวจสอบเงื่อนไข (เหมือนกับใน For Loop)
  console.log(i) // 3. ชุดคำสั่งที่ต้องการทำงาน
  i++ // 4. อัปเดตค่าตัวแปรควบคุม (เขียนไว้ท้ายสุดในบล็อก Loop)
}

ทั้งสองแบบแสดงผล: 0, 1, 2, 3, 4

3. เมื่อใดที่ควรเลือกใช้ While Loop?

  1. เมื่อไม่ทราบจำนวนรอบการทำงานที่แน่นอน

    // ตัวอย่าง: ถามข้อมูลจากผู้ใช้ไปเรื่อย ๆ จนกว่าจะพิมพ์คำว่า "quit" เพื่อออก
let userInput = ""
while (userInput !== "quit") {
  userInput = prompt("Enter a command (or 'quit' to exit):")
  if (userInput !== "quit") {
    console.log(`You entered: ${userInput}`)
  }
}

  2. เมื่อต้องประมวลผลโครงสร้างข้อมูลแบบเชื่อมโยงที่ซ้อนกัน (Linked Data Structures)

    const person = {
  name: "Kyle",
  friend: {
    name: "Joe",
    friend: {
      name: "Sally",
      friend: null, // Sally ไม่มีเพื่อนต่อท้ายแล้ว
    },
  },
}

// ค้นหาเพื่อนคนสุดท้ายในสายความสัมพันธ์ (Linked List)
let currentPerson = person
while (currentPerson.friend !== null) {
  currentPerson = currentPerson.friend
  console.log(`Current friend: ${currentPerson.name}`)
}

console.log(`Final friend: ${currentPerson.name}`) // "Final friend: Sally"

    กรณีนี้การใช้ For Loop จะทำได้ยากมาก เนื่องจากเราไม่มีทางรู้ล่วงหน้าเลยว่าสายความสัมพันธ์นี้จะมีความลึกกี่ชั้น แต่ While Loop สามารถจัดการได้ง่าย ๆ เพียงแค่สั่งให้ทำงานไปเรื่อย ๆ จนกว่าจะเจอค่าที่เป็น null

สรุปแนวทางการเลือกระหว่าง For Loop และ While Loop

ควรเลือกใช้ For Loop เมื่อ:

  • ทราบจำนวนรอบการทำงานที่แน่นอนล่วงหน้า
  • ต้องการวนลูปดึงข้อมูลจาก Array (Array)
  • มีจุดเริ่มต้น จุดสิ้นสุด และขั้นตอนการอัปเดตค่าที่ชัดเจน

ควรเลือกใช้ While Loop เมื่อ:

  • ไม่ทราบจำนวนรอบการทำงานที่แน่นอนล่วงหน้า
  • ต้องทำงานกับโครงสร้างข้อมูลที่ซ้อนกันหรือเชื่อมโยงกันไปเรื่อย ๆ

4. การควบคุมการทำงานด้วย break และ continue

เราสามารถใช้คำสั่ง break (เพื่อหยุดลูปทันที) และ continue (เพื่อข้ามไปรอบถัดไป) ภายใน While Loop ได้เช่นเดียวกับใน For Loop

5. การใช้งาน Do-While Loop

Do-While Loop เป็นลูปที่มีลักษณะพิเศษคือ จะทำงานในบล็อกคำสั่งก่อนอย่างน้อย 1 รอบเสมอ จากนั้นจึงค่อยมาตรวจสอบเงื่อนไขทีหลัง:

let userInput
do {
  userInput = prompt("Enter 'yes' to continue:")
  console.log(`You entered: ${userInput}`)
} while (userInput !== "yes")

💡 ความแตกต่างสำคัญ: ในขณะที่ While Loop ปกติจะตรวจสอบเงื่อนไขก่อนรันโค้ดเสมอ (ถ้าเงื่อนไขเป็นเท็จตั้งแต่แรก โค้ดจะไม่รันเลย) แต่ Do-While Loop จะทำงานก่อน 1 รอบเสมอแล้วค่อยตรวจสอบเงื่อนไขในภายหลัง

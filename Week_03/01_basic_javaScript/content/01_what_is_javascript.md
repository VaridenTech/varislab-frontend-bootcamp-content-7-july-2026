# W03.01 — JavaScript คืออะไร?

เอกสารนี้เป็นโน้ตสำหรับผู้สอน ใช้สอนคาบแรกของ JavaScript ได้ตามลำดับจากบนลงล่าง (ประมาณ 60 นาที)

## เป้าหมายของคลาส

หลังจบคลาสนี้ นักเรียนควรจะสามารถ:

1. อธิบายบทบาทของ JavaScript บนเว็บไซต์ได้
2. แยกหน้าที่ของ HTML, CSS และ JavaScript ได้
3. ยกตัวอย่างงานที่ JavaScript ใช้ได้ นอกเหนือจากหน้าเว็บ
4. เข้าใจภาพรวมของคำว่า `scripting language` โดยยังไม่ต้องจำรายละเอียดเชิงลึก
5. พร้อมเปิด browser console และเริ่มเขียน JavaScript ในบทถัดไป

> เป้าหมายของวันนี้คือ “เห็นภาพ” ไม่ใช่ท่องจำชื่อ framework หรือเขียนโค้ดยาว ๆ ครับ

## โครงสร้างคลาส 60 นาที

| --- | --- | --- |
| 0–10 นาที | Opening + เชื่อมกับ HTML/CSS | รู้ว่า JavaScript เข้ามาเติมส่วนไหนของเว็บ |
| 10–25 นาที | JavaScript ทำอะไรบน browser | เห็นตัวอย่าง interaction ที่คุ้นเคย |
| 25–38 นาที | JavaScript ใช้ที่ไหนได้บ้าง | เห็นเส้นทางงานที่หลากหลาย |

| 48–55 นาที | กิจกรรมกลุ่มสั้น | เชื่อมความรู้กับเว็บไซต์และแอปจริง |
| 55–60 นาที | เช็กความเข้าใจ + ปิดคลาส | ย้ำสิ่งสำคัญและปูไปบทถัดไป |

## Opening Script

พูดกับนักเรียน:

> ก่อนหน้านี้เราใช้ HTML สร้างโครงสร้างของหน้าเว็บ และใช้ CSS ทำให้หน้าเว็บดูดีและจัดวางเป็นระเบียบ
>
> แต่ถ้าเว็บมีแค่ HTML กับ CSS ปุ่มส่วนใหญ่จะยัง “กดแล้วไม่เกิดอะไรขึ้น” วันนี้เราจะเริ่มเรียนเครื่องมือที่ทำให้เว็บตอบสนองต่อผู้ใช้ได้ นั่นคือ JavaScript

เขียนบนกระดานหรือสไลด์:

```text
HTML = โครงสร้าง
CSS = หน้าตาและการจัดวาง
JavaScript = พฤติกรรมและการโต้ตอบ
```

ใช้ตัวอย่างที่นักเรียนคุ้นเคย:

- กดปุ่ม “เพิ่มลงตะกร้า” แล้วจำนวนสินค้าเพิ่ม
- กรอกรหัสผ่านผิดแล้วมีข้อความเตือน
- กดเมนูบนมือถือแล้วเมนูเลื่อนออกมา
- กดถูกใจแล้วตัวเลขยอดไลก์เปลี่ยนทันที

ถามนักเรียน:

> จากตัวอย่างเหล่านี้ อะไรคือ “หน้าตา” ที่ CSS ช่วยทำ และอะไรคือ “การตอบสนอง” ที่ JavaScript ช่วยทำ?

รับคำตอบ 2–3 คน แล้วสรุปว่า HTML, CSS และ JavaScript มักทำงานร่วมกัน ไม่ได้แข่งขันกัน

## 1. JavaScript ทำอะไรให้เว็บไซต์ได้บ้าง?

JavaScript เป็นภาษาโปรแกรมที่เริ่มต้นจากการทำให้หน้าเว็บมีชีวิตและโต้ตอบกับผู้ใช้ได้ ปัจจุบัน browser มี JavaScript engine อยู่ภายใน จึงสามารถรับและรันโค้ด JavaScript ได้

### ตัวอย่างที่ควรอธิบาย

ลองให้ทุกคนดูปุ่มนี้ในเชิงแนวคิด:

```html
<button id="welcomeButton">กดที่นี่</button>
<p id="message">ยังไม่ได้กดปุ่ม</p>
```

เมื่อผู้ใช้กดปุ่ม JavaScript สามารถเปลี่ยนข้อความบนหน้าได้:

```js
const button = document.querySelector("#welcomeButton")
const message = document.querySelector("#message")

button.addEventListener("click", () => {
  message.textContent = "ยินดีต้อนรับสู่ JavaScript!"
})
```

วิธีอธิบายในคลาส:

> ตอนนี้ยังไม่ต้องจำทุกบรรทัดครับ ให้ดูภาพรวมก่อน: เราเลือกปุ่ม รอฟังเหตุการณ์ `click` และเมื่อเกิดการคลิก ก็เปลี่ยนข้อความในหน้าเว็บ

ย้ำกับนักเรียนว่า JavaScript ทำได้มากกว่าการเปลี่ยนข้อความ เช่น:

- ตรวจว่าผู้ใช้กรอก form ครบหรือไม่
- แสดงหรือซ่อนเมนู, modal และ password
- ขอข้อมูลใหม่จาก server แล้วอัปเดตหน้าโดยไม่ต้อง reload ทั้งหน้า
- คำนวณราคา คะแนน หรือยอดรวมสินค้า
- สร้างเกมและ animation บน browser

### จุดที่ควรย้ำ

อย่าพูดว่า JavaScript “ทำให้เว็บสวย” เพราะงานหลักนั้นเป็น CSS

ให้ใช้ประโยคนี้แทน:

> CSS บอกว่า element ควรหน้าตาอย่างไร ส่วน JavaScript บอกว่าเมื่อผู้ใช้ทำอะไรขึ้นมา หน้าเว็บควรตอบสนองอย่างไร

## 2. JavaScript ใช้งานได้ที่ไหนบ้าง?

แม้ JavaScript จะดังจากงานหน้าเว็บ แต่โค้ดเดียวกันไม่จำเป็นต้องรันใน browser เท่านั้น ภาพรวมที่อยากให้นักเรียนเห็นมี 4 พื้นที่ดังนี้

### Browser / Frontend

นี่คือจุดเริ่มต้นของเราในคอร์สนี้ JavaScript ทำงานใน browser ของผู้ใช้ เพื่อควบคุมสิ่งที่เห็นและสิ่งที่กดบนหน้าเว็บ

ตัวอย่าง: ปุ่ม, form, เมนู, carousel, dashboard และข้อมูลที่เปลี่ยนทันที

### Server / Backend

JavaScript สามารถรันบนเครื่อง server ผ่าน runtime เช่น `Node.js` ได้ `Node.js` เป็นตัวเลือกที่ใช้กันแพร่หลายมากสำหรับงาน backend

บน server เราอาจใช้ JavaScript เพื่อ:

- สร้าง API ให้ frontend เรียกใช้
- รับข้อมูลจาก form และบันทึกลงฐานข้อมูล
- ตรวจสิทธิ์ผู้ใช้ เช่น login
- ประมวลผลข้อมูลก่อนส่งกลับไปยัง browser

บอกนักเรียนเพิ่มเติมได้ว่า `Bun` และ `Deno` ก็เป็น runtime ของ JavaScript เช่นกัน แต่วันนี้ให้จำ `Node.js` ชื่อเดียวก็พอ

### Desktop application

เราสามารถสร้างโปรแกรมบนคอมพิวเตอร์ด้วยเทคโนโลยีเว็บและ JavaScript ได้ เช่น `Electron` หรือ `Tauri`

ตัวอย่างโปรแกรมที่นักเรียนอาจเคยใช้ ได้แก่ Visual Studio Code, Slack และ Discord

> ไม่ต้องจำว่าแต่ละโปรแกรมสร้างด้วยอะไรทั้งหมด ประเด็นคือทักษะ JavaScript เปิดทางไปสู่โปรแกรมที่ไม่ได้อยู่ใน browser ด้วย

### Mobile application

JavaScript ยังใช้สร้างแอปมือถือได้ผ่านเครื่องมือ เช่น `React Native`, `Ionic`, `Tauri` หรือ `NativeScript`

เน้นว่า framework เหล่านี้เป็นเครื่องมือที่ค่อยเรียนภายหลัง ตอนนี้เรากำลังสร้างพื้นฐานภาษา JavaScript ที่ใช้ต่อยอดได้ทุกทาง

### ประโยคสรุปสำหรับผู้สอน

> เริ่มจาก JavaScript ในหน้าเว็บก่อน เพราะเห็นผลเร็วที่สุด แต่พื้นฐานที่เราเรียนวันนี้สามารถพาเราไปทำ frontend, backend, desktop และ mobile ได้

### ความเข้าใจผิดที่ควรแก้ทันที

`JavaScript` และ `Java` เป็นคนละภาษา แม้ชื่อจะคล้ายกันมาก เราเรียน JavaScript ไม่ได้แปลว่ากำลังเรียน Java ครับ

## 3. JavaScript เป็น Scripting Language หมายความว่าอย่างไร?

สำหรับผู้เริ่มต้น ให้คิดว่า scripting language คือภาษาที่เราเขียนคำสั่ง แล้วส่งให้ environment ที่รองรับ—เช่น browser หรือ `Node.js`—รันได้อย่างรวดเร็ว

เมื่อเราเขียน JavaScript ใน browser console แล้วกด Enter เรามักเห็นผลทันที เช่น:

```js
console.log("Hello, JavaScript!")
```

### คุณสมบัติที่ควรรู้ในระดับนี้

- **Interpreted / รันผ่าน engine** — JavaScript engine รับโค้ดและทำให้มันทำงาน เราจึงทดลองคำสั่งสั้น ๆ ได้สะดวก
- **Dynamic** — ชนิดของข้อมูลไม่ต้องประกาศล่วงหน้าทุกครั้ง เช่นตัวแปรหนึ่งอาจเก็บข้อความหรือจำนวนได้ แต่เรื่องนี้จะเรียนอย่างละเอียดในบทต่อ ๆ ไป
- **High-level** — ภาษาออกแบบมาให้เราโฟกัสกับปัญหาที่ต้องการแก้ เช่นจัดการปุ่มหรือข้อมูล โดยไม่ต้องควบคุมรายละเอียดระดับเครื่องคอมพิวเตอร์ทั้งหมด

### ข้อควรพูดให้ถูกต้อง

ในคาบแรก เราอาจเรียก JavaScript ว่า “ภาษาที่รันได้ทันที” เพื่อให้เห็นภาพ แต่ไม่ควรสรุปว่า JavaScript ไม่มีขั้นตอน compile เลย

พูดกับนักเรียนได้ว่า:

> สิ่งที่เราเขียนถูกส่งให้ JavaScript engine รัน และ engine สมัยใหม่อาจแปลงหรือปรับโค้ดให้ทำงานเร็วขึ้นภายในเอง วันนี้เราไม่จำเป็นต้องรู้กลไกนั้นละเอียด แค่รู้ว่าเราลองโค้ดและเห็นผลได้เร็ว

## 4. ทำไม JavaScript จึงเหมาะกับการเริ่มเขียนโปรแกรมบนเว็บ?

JavaScript เหมาะกับคลาสแรก เพราะนักเรียนมีสิ่งที่ใช้รันโค้ดอยู่แล้ว นั่นคือ browser

ข้อดีที่ควรย้ำ:

1. **เห็นผลเร็ว** — เขียนไม่กี่บรรทัดก็เห็นข้อความหรือหน้าเว็บเปลี่ยน
2. **ทดลองได้ง่าย** — ใช้ browser console เพื่อทดสอบแนวคิดสั้น ๆ ได้
3. **debug ได้สะดวก** — browser มี DevTools ช่วยดู error, `console.log()` และตรวจ element
4. **ต่อยอดได้กว้าง** — จากเว็บเล็ก ๆ ไปสู่ระบบ backend แอปมือถือ หรือโปรแกรม desktop ได้

ถามชวนคิด:

> ถ้าเราเปลี่ยนข้อความหลังจากกดปุ่มได้แล้ว เราน่าจะสร้าง feature อะไรต่อได้บ้าง?

ตัวอย่างคำตอบ: ปุ่มเพิ่มสินค้า, ระบบนับคะแนน, แบบฟอร์มสมัครสมาชิก หรือเกมตอบคำถาม

## กิจกรรม 5 นาที: มองหา JavaScript รอบตัว

### วิธีทำ

1. แบ่งนักเรียนเป็นคู่ หรือให้คิดคนเดียว
2. ให้เลือกเว็บไซต์หรือแอปหนึ่งอย่างที่ใช้ทุกวัน เช่น YouTube, Facebook, Shopee, Google Maps หรือเว็บไซต์ธนาคาร
3. ให้ตอบคำถาม 3 ข้อนี้ลงในโน้ต:
   - ผู้ใช้ “ทำอะไร” บนหน้านั้นได้บ้าง?
   - เมื่อผู้ใช้กดหรือพิมพ์แล้ว หน้าเว็บ “ตอบสนอง” อย่างไร?
   - จากสิ่งที่เห็น ส่วนไหนน่าจะเป็นงานของ JavaScript?
4. ให้ 2–3 คู่แชร์คำตอบหน้าห้อง

### เฉลยแนวคิด

ไม่มีคำตอบเดียว ให้ฟังเหตุผลของนักเรียนแล้วเชื่อมกลับมาว่า JavaScript มักเกี่ยวกับการรับ input, ตัดสินใจ, เปลี่ยนข้อมูล และอัปเดตสิ่งที่ผู้ใช้เห็น

## คำถามเช็กความเข้าใจ

ใช้ถามปากเปล่าหรือให้ตอบใน chat ก่อนจบคาบ:

1. HTML, CSS และ JavaScript มีหน้าที่ต่างกันอย่างไร?
2. ยกตัวอย่าง interaction หนึ่งอย่างที่ JavaScript ช่วยทำบนเว็บไซต์
3. นอกจาก browser แล้ว JavaScript รันบนที่ใดได้อีกหนึ่งที่?
4. `Node.js` เกี่ยวข้องกับ JavaScript อย่างไร?
5. JavaScript กับ Java คือภาษาเดียวกันหรือไม่?

### คำตอบที่ผู้สอนควรได้ยินโดยประมาณ

- HTML คือโครงสร้าง, CSS คือหน้าตา/การจัดวาง, JavaScript คือพฤติกรรมและการโต้ตอบ
- ตัวอย่างเช่น ปุ่ม, form validation, เมนู หรือการอัปเดตข้อมูล
- Server, desktop application หรือ mobile application
- `Node.js` คือ runtime ที่ทำให้ JavaScript รันนอก browser โดยนิยมใช้กับงาน server
- ไม่ใช่ เป็นคนละภาษา

## สรุปก่อนจบคลาส

พูดปิดคาบ:

> วันนี้เราเริ่มเห็นแล้วว่า JavaScript คือภาษาโปรแกรมที่ทำให้เว็บโต้ตอบกับผู้ใช้ได้ และยังต่อยอดไปยัง server, desktop และ mobile ได้ด้วย
>
> แต่สิ่งสำคัญที่สุดในตอนนี้คือ: JavaScript ทำให้เราสั่งให้หน้าเว็บ “ทำงาน” ได้ ไม่ใช่แค่แสดงผลอยู่เฉย ๆ

สรุปบนกระดานอีกครั้ง:

```text
HTML สร้างสิ่งที่อยู่บนหน้า
CSS จัดหน้าตาและตำแหน่ง
JavaScript ทำให้สิ่งนั้นตอบสนองต่อผู้ใช้
```

## สะพานไปยังบทถัดไป

คาบถัดไป เราจะเริ่มรัน JavaScript ด้วยตัวเองและเขียนคำสั่งแรกใน browser console

บอกนักเรียน:

> วันนี้เรารู้แล้วว่า JavaScript มีไว้ทำอะไร ต่อไปเราจะเริ่มสั่งมันจริง ๆ และดูผลลัพธ์ของโค้ดตัวแรกด้วยตาของเราเอง

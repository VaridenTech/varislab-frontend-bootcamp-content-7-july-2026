การเลือกองค์ประกอบหน้าเว็บ (Selecting Elements)

ทักษะพื้นฐานที่สำคัญที่สุดของการพัฒนาเว็บคือ การเลือกองค์ประกอบ HTML (HTML Elements) เพื่อนำมาควบคุม สั่งงาน หรือเปลี่ยนแปลงข้อมูลด้วย JavaScript โดย Browser มี Method สำเร็จรูปให้เราเลือกใช้งานตามความเหมาะสมดังนี้:

วิธีที่ 1: การเลือกด้วย ID (document.getElementById())

ใช้สำหรับเลือกองค์ประกอบ HTML เพียง ตัวเดียว โดยอ้างอิงจาก Attribute id ที่ระบุไว้:

<div id="header">This is a header</div>
const header = document.getElementById("header")
console.log(header)
header.style.color = "blue"

คุณสมบัติสำคัญ:

  • จะส่งคืน Object HTML Element กลับมาเพียงตัวเดียว หรือส่งคืน null หากไม่พบ ID นั้นในหน้าเว็บ
  • ค่า ID ควรจะมีความเฉพาะเจาะจงและไม่ซ้ำกันเลยในหน้าเว็บนั้น ๆ
  • ในการส่ง Argument ให้ระบุเฉพาะชื่อ ID ตรง ๆ (ไม่ต้องใส่เครื่องหมาย # นำหน้า)
  • เป็น Method ที่ทำงานได้รวดเร็วและมีประสิทธิภาพสูงที่สุดในระบบ
// ❌ วิธีที่ผิด (ห้ามใส่เครื่องหมาย #)
document.getElementById("#header")

// ✅ วิธีที่ถูกต้อง
document.getElementById("header")

วิธีที่ 2: การเลือกด้วยชื่อ Class (document.getElementsByClassName())

ใช้สำหรับเลือกองค์ประกอบ HTML ทั้งหมด ที่มีชื่อ Class ตรงกับที่เรากำหนด:

<div class="content">First</div>
<div class="content">Second</div>
<div class="content">Third</div>
const contentDivs = document.getElementsByClassName("content")
console.log(contentDivs.length) // 3
console.log(contentDivs[0])

การจัดการข้อมูลประเภท HTMLCollection

Method นี้จะส่งคืนข้อมูลกลับมาในรูปแบบ HTMLCollection ซึ่งมีลักษณะคล้าย Array (มี .length และเข้าถึงด้วย index ได้) แต่ ไม่ใช่ Array จริง ๆ ทำให้ไม่สามารถใช้ Method จัดการ Array ทั่วไป (เช่น forEach, map) ได้โดยตรง:

// วิธีที่ 1: แปลง HTMLCollection ให้เป็น Array จริงก่อนนำไปใช้งาน
Array.from(contentDivs).forEach((div) => div.style.color = "green")

// วิธีที่ 2: ใช้ For Loop แบบดั้งเดิมในการวนลูปเข้าถึงข้อมูล
for (let i = 0; i < contentDivs.length; i++) {
  contentDivs[i].style.backgroundColor = "lightgray"
}
contentDivs[0].style.fontWeight = "bold"

วิธีที่ 3: การเลือกด้วย CSS Selector ตัวแรก (document.querySelector()) — แนะนำ

เป็น Method ที่ยืดหยุ่นและแนะนำให้ใช้งานเป็นหลัก โดยเราสามารถระบุเงื่อนไขการเลือกในรูปแบบของ CSS Selector ได้เลย และระบบจะส่งคืนองค์ประกอบ ตัวแรกสุด ที่ตรงตามเงื่อนไขกลับมาเพียงตัวเดียว:

const header = document.querySelector("#header")
const firstContent = document.querySelector(".content")
const firstDiv = document.querySelector("div")
const submitButton = document.querySelector('[data-action="submit"]')

วิธีที่ 4: การเลือกด้วย CSS Selector ทั้งหมด (document.querySelectorAll()) — แนะนำ

ใช้สำหรับเลือกองค์ประกอบ HTML ทั้งหมด ที่ตรงตาม CSS Selector ที่เรากำหนด โดยจะส่งคืนข้อมูลกลับมาในรูปแบบของ NodeList:

const allContent = document.querySelectorAll(".content")
const allDivs = document.querySelectorAll("div")
const allInputs = document.querySelectorAll("input")

การจัดการข้อมูลประเภท NodeList

แม้ว่า NodeList จะไม่ใช่ Array แท้ ๆ เช่นกัน แต่ Browser สมัยใหม่ได้ออกแบบให้ NodeList มี Method forEach() ติดมาให้ในตัว ทำให้เราสามารถวนลูปจัดการข้อมูลได้ทันทีโดยไม่ต้องแปลงชนิดข้อมูลก่อน:

document.querySelectorAll(".content").forEach((div) => {
  div.style.padding = "10px"
})

5. ความแตกต่างระหว่าง Live Collection และ Static Collection

ความแตกต่างสำคัญที่ซ่อนอยู่คือ:
1. getElementsByClassName จะส่งคืนข้อมูลแบบ Live Collection หมายความว่าหากมีการเพิ่มหรือลบ Element บนหน้าเว็บในภายหลัง ข้อมูลในตัวแปรนี้จะอัปเดตตามโดยอัตโนมัติ
2. querySelectorAll จะส่งคืนข้อมูลแบบ Static Collection หมายความว่าข้อมูลจะเป็นภาพถ่าย ณ ขณะที่เรียกใช้งานเท่านั้น หากมีการเปลี่ยนแปลงโครงสร้างหน้าเว็บในภายหลัง ข้อมูลในตัวแปรจะไม่เปลี่ยนตามจนกว่าจะสั่งเรียก Method ใหม่อีกครั้ง

const liveCollection = document.getElementsByClassName("content")
const newDiv = document.createElement("div")
newDiv.className = "content"
document.body.appendChild(newDiv)
console.log(liveCollection.length) // จำนวนสมาชิกจะอัปเดตเพิ่มขึ้นโดยอัตโนมัติ
const staticNodeList = document.querySelectorAll(".content")
// หลังเพิ่ม Element ค่า length ยังคงเดิม ไม่เปลี่ยนแปลงตาม

💡 คำแนะนำ: ในการพัฒนาเว็บทั่วไป เรามักจะต้องการข้อมูลแบบ Static (จาก querySelectorAll) มากกว่า เพราะช่วยลดพฤติกรรมแปลก ๆ ของโค้ดและทำให้ควบคุมทิศทางของโปรแกรมได้ง่ายกว่า

6. สรุปแนวทางการเลือกใช้งาน

คำแนะนำหลัก:
1. ในกรณีทั่วไป แนะนำให้ใช้ querySelector และ querySelectorAll เป็นหลัก เนื่องจากมีความยืดหยุ่นสูงมากและใช้รูปแบบ CSS Selector ที่เราคุ้นเคยอยู่แล้ว
2. หากต้องการเลือกองค์ประกอบเดียวที่มี ID เจาะจงและต้องการความรวดเร็วในการประมวลผลสูงสุด ให้เลือกใช้ getElementById

7. ตัวอย่างการเขียน CSS Selector ขั้นสูงร่วมกับ querySelector

document.querySelector('input[name="username"]')
document.querySelectorAll('a[href^="https://"]')
document.querySelector("li:first-child")
document.querySelector("div:not(.hidden)")
document.querySelector("nav > ul > li")
document.querySelector("h2 + p")
document.querySelector("article p")
document.querySelector("button, input[type='submit']")

ลองทำแบบฝึกหัด

จงเขียนโค้ด JavaScript เพื่อจัดการหน้าเว็บดังต่อไปนี้:
1. เปลี่ยนข้อความใน Header (ID: main-header) ให้เป็นคำว่า "Welcome!"
2. ปรับเปลี่ยนสีตัวอักษรของ Paragraph (<p>) ทั้งหมดในหน้าเว็บให้เป็นสีน้ำเงิน
3. ซ่อน Paragraph ตัวสุดท้ายของหน้าเว็บ (ไม่ให้แสดงผล)
4. เพิ่มเส้นขอบสีแดงขนาด 2px ให้กับช่องกรอกข้อมูล (<input>) ทั้งหมดที่มี Class required

ดูเฉลย 
document.querySelector("#main-header").textContent = "Welcome!"
document.querySelectorAll("p").forEach((p) => p.style.color = "blue")
document.querySelector("p:last-child").style.display = "none"
document.querySelectorAll("input.required").forEach((input) => {
  input.style.border = "2px solid red"
})

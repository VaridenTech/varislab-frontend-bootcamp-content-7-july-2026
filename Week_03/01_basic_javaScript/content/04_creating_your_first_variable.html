การสร้างตัวแปรแรกของคุณ (Variables)

ตัวแปร (Variables) เปรียบเสมือนกล่องเก็บของที่มีป้ายกำกับติดไว้ ซึ่งใช้สำหรับจัดเก็บข้อมูลที่คุณต้องการนำกลับมาใช้ใหม่ในภายหลัง

ตัวแปรคืออะไร?

  • กล่องสำหรับเก็บข้อมูล - ตัวแปรสามารถเก็บค่าต่าง ๆ ได้ เช่น ตัวเลข ข้อความ หรือข้อมูลประเภทอื่น ๆ
  • การอ้างอิงที่นำกลับมาใช้ซ้ำได้ - เมื่อคุณสร้างตัวแปรขึ้นมาแล้ว คุณสามารถเรียกใช้ชื่อของมันเพื่อเข้าถึงข้อมูลข้างในได้จากทุกที่ในโค้ด
  • พื้นที่จัดเก็บที่ยืดหยุ่น - ค่าที่อยู่ภายในตัวแปรสามารถอัปเดตหรือเปลี่ยนแปลงได้ตลอดเวลาในขณะที่โปรแกรมกำลังทำงาน

การประกาศตัวแปร (Declaring Variables)

ในภาษา JavaScript มี 3 วิธีในการประกาศตัวแปร แต่ในบทนี้เราจะเน้นไปที่วิธีที่นิยมและปลอดภัยที่สุด นั่นคือการใช้ Keyword let

let variableOne
let name = "Sally"
let age = 25

ประเด็นสำคัญที่ควรรู้:

  • ไม่จำเป็นต้องกำหนดค่าทันที - คุณสามารถประกาศตัวแปรทิ้งไว้ก่อนโดยยังไม่ต้องกำหนดค่าเริ่มต้นให้มันได้
  • เครื่องหมายเท่ากับตัวเดียว (=) คือตัวดำเนินการสำหรับกำหนดค่า (Assignment Operator) ไม่ใช่การเปรียบเทียบว่าเท่ากัน
  • การตั้งชื่อตัวแปร ควรตั้งชื่อให้สื่อความหมายอย่างชัดเจนว่าเก็บข้อมูลอะไรไว้
  • ตัวพิมพ์เล็กและตัวพิมพ์ใหญ่มีความแตกต่างกัน (Case-Sensitive) - ตัวแปรชื่อ age และ Age จะถือว่าเป็นตัวแปรคนละตัวกัน

การอัปเดตค่าของตัวแปร (Updating Variables)

จุดเด่นที่สำคัญของตัวแปรคือ คุณสามารถอัปเดตหรือเปลี่ยนค่าที่อยู่ภายในตัวแปรได้ตลอดเวลา:

let score = 10
console.log(score) // ผลลัพธ์: 10

score = 15
console.log(score) // ผลลัพธ์: 15

score = "name"
console.log(score) // ผลลัพธ์: name

ประโยชน์ของการใช้งานตัวแปร

  • ทำให้อ่านโค้ดง่ายขึ้น (Readability): ช่วยให้ผู้อื่น (หรือตัวคุณเองในอนาคต) เข้าใจได้ทันทีว่าข้อมูลนี้คืออะไร
  • บำรุงรักษาโค้ดง่าย (Maintainability): หากต้องการแก้ไขค่าข้อมูล คุณสามารถแก้ไขที่ตัวแปรเพียงจุดเดียว โดยไม่ต้องตามไปแก้ในทุก ๆ บรรทัด
  • นำกลับมาใช้ซ้ำได้ (Reusability): สามารถเรียกใช้ข้อมูลเดิมซ้ำ ๆ ได้สะดวกผ่านชื่อตัวแปร

แนวทางปฏิบัติที่ดีในการตั้งชื่อตัวแปร (Best Practices)

  • ใช้ชื่อที่สื่อความหมายชัดเจน: หลีกเลี่ยงการใช้ตัวอักษรตัวเดียว เช่น ใช้ age แทน a หรือใช้ userName แทน u
  • ใช้รูปแบบ camelCase: ใน JavaScript นิยมเขียนชื่อตัวแปรแบบอักษรตัวแรกพิมพ์เล็ก และคำถัดไปขึ้นต้นด้วยตัวพิมพ์ใหญ่ เช่น firstName แทน first_name
  • หลีกเลี่ยงการเปลี่ยนชนิดข้อมูลสลับไปมา: แม้ JavaScript จะยอมให้เปลี่ยนจากตัวเลขเป็นข้อความได้ แต่ในทางปฏิบัติที่ดี ควรให้ตัวแปรนั้นเก็บข้อมูลชนิดเดิมตลอดเพื่อป้องกันความสับสน

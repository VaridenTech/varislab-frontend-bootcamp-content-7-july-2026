NaN (ไม่ใช่ตัวเลข)

NaN เป็นค่าพิเศษใน JavaScript ย่อมาจาก “Not a Number”

อะไรทำให้เกิด NaN?

NaN เกิดขึ้นเมื่อคุณพยายามแปลงตัวแปรที่ไม่ใช่ตัวเลข (เช่น String) ให้เป็น Number:

const result = parseInt("hello")

console.log(result) // NaN
console.log(typeof result) // "number"

แม้ NaN จะแปลว่า “ไม่ใช่ตัวเลข” แต่ชนิดข้อมูลของมันคือ "number"

การตรวจสอบ NaN

คุณอาจคิดว่าสามารถตรวจสอบ NaN แบบนี้ได้ แต่มันใช้ไม่ได้:

const a = parseInt("hello")

console.log(a == NaN) // false

ทำไมจึงใช้ไม่ได้? NaN จะไม่เท่ากับค่าใดเลย แม้แต่ตัวมันเอง!

console.log(NaN == NaN) // false

วิธีที่ถูกต้อง: isNaN()

หากต้องการตรวจสอบว่าค่าเป็น NaN หรือไม่ ให้ใช้ฟังก์ชัน isNaN() ที่มีมาให้:

const a = parseInt("hello") // NaN
const b = 5 // number

console.log(isNaN(a)) // true (a เป็น NaN)
console.log(isNaN(b)) // false (b เป็นตัวเลขที่ถูกต้อง)

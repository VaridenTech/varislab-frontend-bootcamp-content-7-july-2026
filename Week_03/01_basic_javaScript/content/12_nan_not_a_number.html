NaN (Not a Number)

NaN เป็นค่าพิเศษใน JavaScript ซึ่งย่อมาจาก “Not a Number” (ไม่ใช่ตัวเลข)

อะไรทำให้เกิดค่า NaN?

ค่า NaN มักจะเกิดขึ้นเมื่อเราพยายามแปลงข้อมูลที่ไม่ใช่ตัวเลข (เช่น ข้อความทั่วไป) ให้กลายเป็นตัวเลข (Number) แต่ตัวแปรนั้นไม่สามารถแปลงได้:

const result = parseInt("hello")

console.log(result) // NaN
console.log(typeof result) // "number"

💡 เกร็ดความรู้: แม้ว่า NaN จะย่อมาจาก "ไม่ใช่ตัวเลข" แต่ในทางเทคนิค ชนิดข้อมูล (Type) ของมันกลับเป็น "number"!

วิธีการตรวจสอบค่า NaN

คุณอาจจะคิดว่าเราสามารถใช้ตัวดำเนินการเปรียบเทียบทั่วไปเพื่อตรวจสอบค่า NaN ได้ แต่มันไม่สามารถใช้งานได้:

const a = parseInt("hello")

console.log(a == NaN) // false

ทำไมจึงเปรียบเทียบแบบนั้นไม่ได้? เพราะในภาษา JavaScript ค่า NaN มีคุณสมบัติพิเศษคือ มันจะไม่เท่ากับค่าใด ๆ เลยในโลกนี้ แม้กระทั่งตัวมันเอง!

console.log(NaN == NaN) // false

วิธีการที่ถูกต้อง: ใช้ Function isNaN()

หากต้องการตรวจสอบว่าตัวแปรใดมีค่าเป็น NaN หรือไม่ วิธีเดียวที่ถูกต้องคือการใช้ Function สำเร็จรูปที่ชื่อว่า isNaN():

const a = parseInt("hello") // NaN
const b = 5 // number

console.log(isNaN(a)) // true (a เป็น NaN จริง)
console.log(isNaN(b)) // false (b เป็นตัวเลขปกติ ไม่ใช่ NaN)

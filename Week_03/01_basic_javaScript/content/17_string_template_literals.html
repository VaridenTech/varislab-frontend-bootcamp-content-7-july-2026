String Template Literal

จนถึงตอนนี้เราสร้าง String ด้วยเครื่องหมายคำพูดเดี่ยว '' หรือคู่ "" แต่ JavaScript มีวิธีที่สามซึ่งทรงพลังกว่ามาก คือ template literal ที่ใช้ backtick ``

ไวยากรณ์พื้นฐานของ Template Literal

Template literal ใช้ backtick แทนเครื่องหมายคำพูดปกติ (ปุ่มเหนือ Tab บนแป้นพิมพ์ส่วนใหญ่):

const a = `Hi`
console.log(a) // Hi

ทำงานเหมือน String ปกติทุกประการ แต่ template literal มีความสามารถพิเศษ

การฝังตัวแปรด้วย ${}

พลังที่แท้จริงของ template literal คือการฝังตัวแปรไว้ใน String โดยตรงด้วยไวยากรณ์ ${}:

const firstName = "Kyle"
const lastName = "Cook"

console.log(firstName + " " + lastName) // Kyle Cook
console.log(`${firstName} ${lastName}`) // Kyle Cook

ไวยากรณ์ ${} บอก JavaScript ว่า “รันโค้ดภายในวงเล็บนี้แล้วนำผลลัพธ์มาใส่ตรงนี้”

การฝังนิพจน์ JavaScript ใด ๆ

คุณสามารถใส่นิพจน์ JavaScript ใด ๆ ภายใน ${} ไม่ใช่แค่ตัวแปร:

const firstName = "Kyle"

console.log(`${firstName} ${2 + 3}`) // Kyle 5
console.log(`Hello ${firstName.toUpperCase()}`) // Hello KYLE
console.log(`Random number: ${Math.random()}`) // Random number: 0.234...

String หลายบรรทัด

Template literal ยังช่วยให้สร้าง String หลายบรรทัดได้ง่าย:

const poem = `Roses are red,
Violets are blue,
Template literals are awesome,
And so are you!`

console.log(poem)
// Roses are red,
// Violets are blue,
// Template literals are awesome,
// And so are you!

เมื่อใดควรใช้ Template Literal

ใช้ template literal เมื่อ:

  • ✅ ต้องฝังตัวแปรใน String
  • ✅ ต้องการ String หลายบรรทัด

ใช้เครื่องหมายคำพูดปกติเมื่อ:

  • ✅ มี String คงที่แบบง่าย ๆ ที่ไม่มีตัวแปร

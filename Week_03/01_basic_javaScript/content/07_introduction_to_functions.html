ความรู้เบื้องต้นเกี่ยวกับ Function

Function คือบล็อกของโค้ดที่ใช้สำหรับจัดเก็บตรรกะหรือชุดคำสั่งต่าง ๆ เอาไว้ เพื่อเรียกใช้งานในภายหลัง เปรียบเสมือนตัวแปรที่ใช้สำหรับเก็บชุดคำสั่งแทนการเก็บค่าข้อมูล

ทำไมเราจึงต้องใช้ Function?

Function ช่วยให้เราหลีกเลี่ยงการเขียนโค้ดเดิมซ้ำ ๆ (Don't Repeat Yourself - DRY) ลองดูตัวอย่างนี้:

console.log("Hello")

// ... มีโค้ดส่วนอื่น ๆ ทำงานอยู่ตรงนี้ ...

console.log("Hello") // ❌ เรากำลังเขียนโค้ดเดิมซ้ำ!

หากในอนาคตเราต้องการเปลี่ยนข้อความจาก "Hello" เป็น "Hi" เราจะต้องตามไปแก้ไขโค้ดในทุก ๆ จุดด้วยตัวเอง ซึ่งเสี่ยงต่อการเกิดข้อผิดพลาดและเสียเวลามาก นี่คือเหตุผลว่าทำไมเราจึงควรใช้ Function

การประกาศ Function (Declaring Functions)

นี่คือโครงสร้างพื้นฐานในการเขียน Function ใน JavaScript:

function sayHi() {
  console.log("Hi")
}

รายละเอียดในแต่ละส่วนประกอบด้วย:

  1. function - Keyword พิเศษที่บอกให้ JavaScript ทราบว่าเรากำลังสร้าง Function ใหม่
  2. sayHi - ชื่อของ Function (ใช้หลักการตั้งชื่อแบบเดียวกับตัวแปร คือ camelCase)
  3. () - วงเล็บสำหรับรับค่า Parameter (Parameter) เข้ามาใช้งานใน Function (ตอนนี้ยังว่างอยู่)
  4. {} - วงเล็บปีกกาสำหรับกำหนดขอบเขตของชุดคำสั่งที่จะทำงานภายใต้ Function นี้ (Function Body)

หลักการตั้งชื่อ Function

การตั้งชื่อ Function ที่ดีควรใช้หลักการเดียวกับการตั้งชื่อตัวแปร ดังนี้:

  • เริ่มต้นด้วยตัวอักษรพิมพ์เล็กเสมอ
  • ใช้รูปแบบ camelCase สำหรับชื่อที่ประกอบด้วยคำหลายคำ (เช่น calculateTotal)
  • เลือกใช้คำกริยาหรือชื่อที่สื่อความหมายชัดเจนว่า Function นี้ทำหน้าที่อะไร
function sayHello() {} // ✅ ตัวอย่างการตั้งชื่อที่ดี
function calculateTotal() {} // ✅ ตัวอย่างการตั้งชื่อที่ดี
function getUserAge() {} // ✅ ตัวอย่างการตั้งชื่อที่ดี

การเรียกใช้งาน Function (Calling/Invoking Functions)

การประกาศ Function เป็นเพียงการเขียนชุดคำสั่งเตรียมไว้เท่านั้น โดย Function จะยังไม่ทำงานจนกว่าเราจะทำการ เรียกใช้งาน (Call / Invoke) มัน:

function sayHi() {
  console.log("Hi")
}

sayHi() // 👈 เรียกใช้งาน Function ที่นี่

เราสามารถเรียกใช้งาน Function เดิมซ้ำ ๆ กี่ครั้งก็ได้ตามต้องการ:

sayHi() // ผลลัพธ์: "Hi"
sayHi() // ผลลัพธ์: "Hi" (เรียกซ้ำได้ทันที)

Function ที่รับค่า Parameter (Parameters)

Function จะทำงานได้ยืดหยุ่นและทรงพลังยิ่งขึ้นเมื่อสามารถรับข้อมูลจากภายนอกเข้ามาประมวลผลได้ ข้อมูลนำเข้าเหล่านี้เรียกว่า Parameter (Parameters):

function printName(name) {
  console.log(name)
}

printName("Kyle") // แสดงผล "Kyle"
printName("Sarah") // แสดงผล "Sarah"

ความแตกต่างระหว่าง Parameter (Parameter) กับ Argument (Argument)

  • Parameter (Parameter) คือ ตัวแปรที่เขียนระบุไว้ในวงเล็บตอนสร้าง Function เพื่อรอรับค่าเข้ามาใช้งาน (เช่น name)
  • Argument (Argument) คือ ค่าข้อมูลจริง ๆ ที่เราส่งเข้าไปให้ Function ตอนเรียกใช้งาน (เช่น "Kyle", "Sarah")

In ทางทฤษฎี ทั้งสองคำนี้มีความแตกต่างกันเล็กน้อย แต่ในชีวิตการทำงานจริง นักพัฒนาส่วนใหญ่มักจะใช้สองคำนี้แทนกันในการสนทนาทั่วไป

การรับ Parameter หลายตัว

เราสามารถกำหนดให้ Function รับ Parameter ได้หลายตัว โดยใช้เครื่องหมายจุลภาค (,) คั่นระหว่างตัวแปร:

function sum(a, b) {
  console.log(a + b)
}

sum(1, 2) // แสดงผล 3
sum(5, 7) // แสดงผล 12

การส่งตัวแปรเป็น Argument

เราสามารถนำตัวแปรที่ประกาศไว้ข้างนอก ส่งเข้าไปเป็น Argument ของ Function ได้เช่นกัน:

function add(x, y) {
  console.log(x + y)
}

let num1 = 10
let num2 = 5

add(num1, num2) // แสดงผล 15
add(3, 7) // หรือจะส่งค่าข้อมูลตรง ๆ เข้าไปเลยก็ได้

การส่งค่ากลับด้วย return (Return Values)

ที่ผ่านมา Function ของเราทำหน้าที่เพียงแสดงผลข้อความออกทางหน้าจอเท่านั้น แต่ในความเป็นจริง Function สามารถ ส่งค่าผลลัพธ์กลับออกมา (Return Value) เพื่อให้เรานำค่านั้นไปใช้งานต่อในส่วนอื่น ๆ ของโปรแกรมได้ โดยใช้ Keyword return:

function add(x, y) {
  return x + y
}

let sum = add(2, 3)
console.log(sum) // 5
console.log(add(10, 7)) // 17

เมื่อโปรแกรมทำงานมาถึงคำสั่ง return Function จะหยุดการทำงานในบรรทัดนั้นทันที และส่งค่าผลลัพธ์ที่อยู่หลัง return กลับออกไป ซึ่งเราสามารถนำค่านี้ไปเก็บไว้ในตัวแปรหรือนำไปประมวลผลต่อได้ทันที

ข้อควรรู้: หาก Function ไม่มีการใช้คำสั่ง return Function นั้นจะส่งค่า undefined กลับออกมาโดยอัตโนมัติ

ข้อผิดพลาดทางไวยากรณ์ที่พบบ่อย (Syntax Errors)

  • ลืมใส่วงเล็บหลังชื่อ Function

    // ❌ ผิด - ไม่มีวงเล็บ
function sayHello {
  console.log("Hello")
}

// ✅ ถูกต้อง
function sayHello() {
  console.log("Hello")
}

  • ลืมใส่วงเล็บปีกกาครอบโค้ด

    // ❌ ผิด - ไม่มีวงเล็บปีกกาเปิด
function sayHello()
  console.log("Hello")
}

// ❌ ผิด - ไม่มีวงเล็บปีกกาปิด
function sayHello() {
  console.log("Hello")

// ✅ ถูกต้อง
function sayHello() {
  console.log("Hello")
}

  • ลืมเรียกใช้งาน Function

    // บรรทัดเหล่านี้สร้าง Function แต่ไม่ได้เรียกใช้
function sayHello() {
  console.log("Hello")
}

// คุณต้องเรียกใช้ Function:
sayHello() // ตอนนี้ Function ทำงานแล้ว

สรุปประโยชน์ของการใช้งาน Function

  1. ลดความซ้ำซ้อนของโค้ด (DRY) - เขียนชุดคำสั่งเพียงครั้งเดียว แต่สามารถเรียกใช้งานซ้ำได้ไม่จำกัด
  2. บำรุงรักษาง่าย (Maintainability) - หากต้องการแก้ไขตรรกะการทำงาน สามารถแก้ไขที่ Function จุดเดียว โค้ดทุกจุดที่เรียกใช้จะอัปเดตตามทันที
  3. ช่วยจัดระเบียบโค้ด (Modularity) - ช่วยแบ่งงานหรือปัญหาที่ซับซ้อนออกเป็นส่วนย่อย ๆ ทำให้โค้ดอ่านง่ายและเป็นระบบมากขึ้น

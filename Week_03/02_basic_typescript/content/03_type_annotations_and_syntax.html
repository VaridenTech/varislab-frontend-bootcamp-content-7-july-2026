Type Annotations และไวยากรณ์พื้นฐานของ TypeScript

หัวใจของ TypeScript คือ Type Annotation — ไวยากรณ์สำหรับ "ประกาศ" ว่าตัวแปร Parameter หรือค่าที่คืนกลับจาก Function ควรมี Type อะไร โดยเขียนเครื่องหมาย Colon (:) ตามด้วยชื่อ Type ต่อท้ายสิ่งที่ต้องการระบุ

การกำหนด Type ให้ตัวแปร

let price: number = 129.99;
price = 99.5;      // OK
price = "on sale"; // Error! Type 'string' is not assignable to type 'number'

เมื่อไหร่ก็ตามที่เรากำหนด Type ให้ตัวแปรแล้ว TypeScript จะจดจำไว้และคอยเตือนทันทีที่มีการกำหนดค่าอื่นที่ไม่ตรงกับ Type นั้น ไม่ว่าจะตอนประกาศตัวแปรครั้งแรก หรือตอน Reassign ค่าใหม่ในภายหลัง

กำหนด Type ไว้ก่อน ค่อยใส่ค่าทีหลังก็ได้

let quantity: number;
quantity = 3;        // OK
quantity = "three";  // Error!
quantity = null;      // Error! (ดูเหตุผลในหัวข้อถัดไป)

Primitive Types พื้นฐาน

TypeScript มี Type สำหรับข้อมูลพื้นฐานของ JavaScript ให้ใช้ตรงตัว:

let age: number = 25;
let name: string = "พลอย";
let isMember: boolean = true;

let missing: undefined = undefined;
let empty: null = null;

ข้อสังเกต: null และ undefined ใน TypeScript แบบ Strict Mode จะไม่ถูกยอมรับให้เป็นค่าของ Type อื่นโดยอัตโนมัติ (ต่างจาก JavaScript ที่ยอมให้ทุกตัวแปรเป็น null ได้เสมอ) เราจะพูดถึงวิธีอนุญาตให้ตัวแปรเป็นได้ทั้งค่าปกติและ null ในหัวข้อ Union Types ด้านล่าง

Literal Types: จำกัดให้เป็นได้แค่ค่าที่ระบุไว้เท่านั้น

นอกจากระบุ "ชนิด" กว้าง ๆ อย่าง string เรายังระบุ "ค่าที่อนุญาต" แบบเจาะจงได้เลย เรียกว่า Literal Type:

let status: "pending" | "shipped" | "delivered";
status = "pending";   // OK
status = "shipped";   // OK
status = "cancelled"; // Error! ไม่ใช่หนึ่งในค่าที่กำหนดไว้

เครื่องหมาย | อ่านว่า "หรือ" — ตัวแปร status เป็นได้แค่หนึ่งในสามค่านี้เท่านั้น รูปแบบนี้มีประโยชน์มากเวลาต้องการแทนที่ "Magic String" ด้วยชุดค่าที่ตรวจสอบได้จริง

การกำหนด Type ให้ Function

Function ใน TypeScript ระบุ Type ได้ทั้งที่ Parameter และค่าที่ Return กลับ:

function calculateTotal(price: number, quantity: number): number {
  return price * quantity;
}

calculateTotal(100, 2);      // 200
calculateTotal("100", 2);    // Error! Argument of type 'string' is not assignable

Arrow Function เขียนแบบเดียวกัน โดยใส่ Type ของค่าที่ Return หลังวงเล็บ Parameter:

const greet = (name: string): string => {
  return `สวัสดีครับ ${name}`;
}

greet("คุณสมชาย"); // "สวัสดีครับ คุณสมชาย"
greet(42);          // Error!

การกำหนด Type ให้ Array

const scores: number[] = [88, 92, 76];
scores.push(95);    // OK
scores.push("A+");  // Error! ใส่ String เข้า Array ที่เป็น number[] ไม่ได้

const tags: string[] = ["sale", "new"];
tags.push("popular"); // OK
tags.push(10);         // Error!

การกำหนด Type ให้ Object

let product: { name: string, price: number };
product = { name: "เสื้อยืด", price: 259 };

product.price = 299;    // OK
product.price = "299";  // Error!
product.color = "blue"; // Error! ไม่มี Property ชื่อ color ใน Type ที่กำหนดไว้

สังเกตว่า TypeScript ไม่ได้ตรวจแค่ Type ของแต่ละ Property เท่านั้น แต่ยังตรวจด้วยว่า Object มี Property ครบตามที่ระบุไว้และไม่มีตัวเกิน

Property ที่ไม่บังคับต้องมี (Optional Properties)

ใส่เครื่องหมาย ? ต่อท้ายชื่อ Property เพื่อบอกว่า Property นั้นมีหรือไม่มีก็ได้:

let product: {
  id: number,
  name: string,
  discountCode?: string,
};

product = { id: 1, name: "เสื้อยืด" };                          // OK - ไม่ใส่ discountCode ก็ได้
product = { id: 2, name: "กางเกง", discountCode: "SALE10" };   // OK
product = { name: "หมวก" };                                     // Error! ขาด id ซึ่งจำเป็นต้องมี

Union Types: ให้ตัวแปรเป็นได้มากกว่าหนึ่ง Type

บางครั้งตัวแปรก็มีความจำเป็นต้องเก็บค่าได้มากกว่าหนึ่ง Type จริง ๆ เช่น รหัสสินค้าที่อาจเป็นได้ทั้งตัวเลขหรือรหัสข้อความ:

let productId: number | string;
productId = 1024;     // OK
productId = "SKU-42"; // OK
productId = true;      // Error!

Union Types ยังเป็นวิธีมาตรฐานในการบอกว่าตัวแปรนั้น "เป็นได้ทั้งค่าปกติ หรือไม่มีค่าเลย":

let selectedProduct: string | null;
selectedProduct = "เสื้อยืด"; // OK
selectedProduct = null;        // OK
selectedProduct = undefined;   // Error! null กับ undefined ไม่ใช่ Type เดียวกัน

Type Narrowing: จำกัดขอบเขต Type ก่อนใช้งาน

เมื่อตัวแปรเป็น Union Type เราจะเรียกใช้ Method หรือ Property ที่มีเฉพาะในบาง Type ทันทีไม่ได้ เพราะ TypeScript ไม่รู้ล่วงหน้าว่าตอนรันจริงตัวแปรนั้นจะเป็น Type ไหน 

let heading: HTMLElement | null;
heading = document.getElementById("page-title");

heading.textContent = "ยินดีต้อนรับ"; // Error! heading อาจเป็น null ก็ได้

วิธีแก้คือใช้เงื่อนไข if เพื่อ "แคบขอบเขต (Narrow)" ของ Type ลง — หลังผ่านเงื่อนไขแล้ว TypeScript จะรู้เองว่าเหลือ Type อะไรอยู่ในแต่ละ Branch: 

if (heading === null) {
  console.log("ไม่พบ Element");
} else {
  heading.textContent = "ยินดีต้อนรับ"; // OK - ในนี้ TypeScript รู้ว่า heading ไม่ใช่ null แน่นอน
}

เทคนิคนี้เรียกว่า Type Guard ใช้ตรวจสอบ Type ด้วย typeof ได้เช่นกัน:

function formatId(id: number | string): string {
  if (typeof id === "number") {
    return `#${id.toString().padStart(4, "0")}`; // ในนี้ id เป็น number แน่นอน
  }
  return id.toUpperCase(); // ในนี้ id เป็น string แน่นอน
}

สร้าง Type ของตัวเองด้วย type Alias

เมื่อ Type ที่เขียนซ้ำ ๆ เริ่มยาวขึ้น เราสามารถตั้งชื่อให้มันแล้วนำไปใช้ซ้ำได้ด้วย Keyword type 

type ProductId = number | string;
type OrderStatus = "pending" | "shipped" | "delivered";

type Product = {
  id: ProductId,
  name: string,
  status: OrderStatus,
  discountCode?: string,
};

const item: Product = {
  id: "SKU-42",
  name: "เสื้อยืด",
  status: "pending",
};

type Alias ไม่ได้สร้าง Type ใหม่ในทางเทคนิค แต่เป็นเพียง "ชื่อเรียก" ของ Type ที่เขียนไว้ ทำให้โค้ดสั้นลง อ่านง่ายขึ้น และแก้ไข Type ได้จากจุดเดียว

สรุปไวยากรณ์ที่ได้เรียนในบทนี้

  • let x: Type — กำหนด Type ให้ตัวแปร
  • function f(x: Type): ReturnType — กำหนด Type ให้ Parameter และค่าที่ Return
  • Type[] — Array ที่สมาชิกทุกตัวเป็น Type เดียวกัน
  • { key: Type } — Object ที่มี Shape ตามที่ระบุ, ใช้ ? สำหรับ Property ที่ไม่บังคับ
  • TypeA | TypeB — Union Type ให้ค่าเป็นได้มากกว่าหนึ่ง Type
  • type Name = ... — ตั้งชื่อ Type เพื่อนำไปใช้ซ้ำ

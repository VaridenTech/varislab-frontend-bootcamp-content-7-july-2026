TypeScript คืออะไร และทำไมต้องใช้?

ตลอดสัปดาห์ที่ผ่านมาเราเขียน JavaScript กันมาพอสมควรแล้ว สัปดาห์นี้เราจะมาทำความรู้จักกับ TypeScript (TS) ภาษาที่ทีมพัฒนาจำนวนมากใช้ต่อยอดจาก JavaScript เพื่อเขียนโค้ดที่ผิดพลาดยากขึ้นและอ่านเข้าใจง่ายขึ้น

TypeScript สัมพันธ์กับ JavaScript อย่างไร

สิ่งสำคัญที่สุดที่ต้องเข้าใจก่อนคือ TypeScript ไม่ใช่ภาษาคนละตัวที่แยกขาดจาก JavaScript แต่เป็น Superset ของ JavaScript หมายความว่าโค้ด JavaScript ที่ถูกต้องทุกบรรทัดก็ถือเป็นโค้ด TypeScript ที่ถูกต้องเช่นกัน TypeScript เพียงแค่ เพิ่ม ไวยากรณ์สำหรับระบุ "Type" เข้าไป แล้วใช้เครื่องมือตรวจสอบ Type นั้นก่อนโค้ดจะถูกนำไปรันจริง

หัวข้อ JavaScript TypeScript
File Extension .js .ts
การทำงาน (Execution) Interpreted (รันได้ทันทีใน Browser หรือ Node.js) ต้อง Compile เป็น JavaScript ก่อน แล้วโค้ด JavaScript ที่ได้จึงถูกรัน
ไวยากรณ์ (Syntax) เป็นส่วนหนึ่งของ TypeScript (Subset) ขยายมาจาก JavaScript (Superset)
ระบบ Type (Type System) Dynamic และ Weak (ยืดหยุ่น แต่ตรวจสอบตอนรันจริงเท่านั้น) Static และ Strong (ระบุชัดเจน ตรวจสอบได้ตั้งแต่ก่อนรัน)

ปัญหาที่ TypeScript พยายามแก้

JavaScript เป็นภาษาที่มี Type แบบ Dynamic (Type ถูกกำหนดตอนโค้ดกำลังรัน ไม่ใช่ก่อนรัน) และแบบ Weak (ยอมแปลง Type ให้อัตโนมัติเมื่อเจอค่าที่ไม่ตรงกับที่คาดไว้) คุณสมบัติทั้งสองข้อนี้ทำให้ JavaScript เขียนง่ายและยืดหยุ่นสูง แต่ก็แลกมาด้วยข้อผิดพลาดที่มักโผล่มาตอนโปรแกรมกำลังรันอยู่แล้ว เช่น 

"Cannot read properties of undefined (reading 'name')"
"user.getFullName is not a function"
"Cannot set properties of null"

ข้อผิดพลาดเหล่านี้มักไม่ปรากฏตอนเขียนโค้ด แต่จะโผล่มาให้เห็นตอนผู้ใช้งานจริงกดปุ่มบางอย่าง หรือแอปพลิเคชันทำงานถึงจุดนั้นพอดี เป้าหมายของ TypeScript คือดักจับข้อผิดพลาดประเภทนี้ตั้งแต่ตอนเขียนโค้ด ก่อนจะถูกส่งไปให้ผู้ใช้งานเจอ

TypeScript ทำอะไรได้บ้าง

ตลอดสัปดาห์นี้เราจะได้เห็น TypeScript ในสามบทบาทหลัก:

  • เครื่องมือตรวจสอบ Type (Type Checker) — อ่านโค้ดของเราแล้วเตือนทันทีถ้าเราใช้ค่าผิดชนิด เช่น พยายามบวกเลขกับ Object
  • เครื่องมือแปลงโค้ด (Compiler / Build Tool) — แปลงไฟล์ .ts เป็น .js ธรรมดา เพื่อให้ Browser หรือ Node.js รันได้ (Type ทั้งหมดจะถูกลบออกตอนแปลง)
  • ผู้ช่วยใน Editor (Dev Tool) — เชื่อมกับ Editor อย่าง VS Code เพื่อให้ Autocomplete, คำแนะนำ, และคำเตือน ขึ้นให้เห็นแบบเรียลไทม์ขณะพิมพ์โค้ด

สิ่งที่จะได้เรียนในสัปดาห์นี้

เราจะค่อย ๆ สร้างความเข้าใจ TypeScript ไปทีละขั้น:

  • การเพิ่ม Type ให้ตัวแปร Function และ Object
  • Union Types และการจำกัดขอบเขต Type (Type Narrowing)
  • การสร้างและขยาย Type ของตัวเองด้วย type และ interface
  • การตั้งค่า Compiler ด้วยไฟล์ tsconfig.json
  • Generic Types สำหรับเขียน Type ที่ใช้ซ้ำได้
  • การใช้งาน TypeScript ร่วมกับ Library และ Ecosystem จริง

ข่าวดีคือ เราไม่จำเป็นต้องแปลงทั้งโปรเจกต์เป็น TypeScript ในคราวเดียว เพราะ TypeScript ถูกออกแบบมาให้ค่อย ๆ ย้ายจาก JavaScript มาได้ทีละไฟล์

สิ่งที่ต้องเตรียม

เนื่องจาก TypeScript ทำงานอยู่บน Node.js เราจึงใช้เครื่องมือชุดเดิมกับที่ติดตั้งไว้แล้วจากสัปดาห์ก่อนหน้า ได้แก่ Node.js (พร้อม npm) และ VS Code ยังไม่ต้องติดตั้งอะไรเพิ่มตอนนี้ — บทถัดไปจะพาติดตั้งตัว TypeScript เอง

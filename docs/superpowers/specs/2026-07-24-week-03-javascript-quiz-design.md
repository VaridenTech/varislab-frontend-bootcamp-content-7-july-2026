# แบบออกแบบชุดคำถาม JavaScript บทที่ 13–26

วันที่: 24 กรกฎาคม 2026

## เป้าหมาย

สร้างข้อสอบปรนัยภาษาไทยสำหรับแพลตฟอร์ม e-learning จากเนื้อหาใน `Week_03/01_basic_javaScript/content/13_equality_type_comparisons.html` ถึง `26_array_methods.html` เพื่อวัดความเข้าใจพื้นฐานหลังเรียนจบช่วงดังกล่าว

## ขอบเขต

- จำนวน 15 ข้อ
- ตัวเลือก 4 ตัวต่อข้อ และมีคำตอบที่ถูกต้องเพียงตัวเลือกเดียว
- ระดับพื้นฐาน เน้นความเข้าใจแนวคิด คำศัพท์ และไวยากรณ์หลัก
- มีโจทย์อ่านผลลัพธ์โค้ดสั้น ๆ เฉพาะกรณีที่ตรงไปตรงมา
- ใช้ภาษาไทยที่อ่านง่าย พร้อมคงคำศัพท์และชื่อคำสั่ง JavaScript ภาษาอังกฤษ
- ทุกข้อมีเฉลยและคำอธิบายสั้น ๆ
- ไม่ใช้เนื้อหาที่อยู่นอกบทเรียน 13–26

## การกระจายเนื้อหา

คำถาม 14 ข้อแรกครอบคลุมบทเรียนบทละ 1 ข้อตามหัวข้อต่อไปนี้:

1. Equality และ Type Comparisons
2. Arrays
3. Objects
4. Reference vs Value
5. String Template Literals
6. `this`, Class และ `new`
7. If Statements, Truthy และ Falsy
8. Ternary Operator
9. Switch Statements
10. For Loops
11. While และ Do-While Loops
12. Recursion
13. Short-Circuit Evaluation รวมถึง `??` และ `?.`
14. Array Methods

ข้อที่ 15 เพิ่มการวัดความเข้าใจ Array Methods โดยเน้นการเลือกใช้ `map()` หรือ `filter()` ซึ่งเป็นทักษะพื้นฐานที่ผู้เรียนมีโอกาสนำไปใช้บ่อย

## รูปแบบแต่ละข้อ

แต่ละข้อประกอบด้วย:

1. หมายเลขข้อและคำถาม
2. โค้ดตัวอย่าง หากจำเป็นต่อคำถาม
3. ตัวเลือก ก.–ง.
4. เฉลย โดยระบุตัวเลือกและข้อความคำตอบ
5. คำอธิบาย 1–2 ประโยค เชื่อมโยงกับหลักการจากบทเรียน

## หลักการออกตัวเลือก

- ตัวเลือกผิดต้องดูเป็นไปได้สำหรับผู้เริ่มต้น แต่ไม่กำกวม
- หลีกเลี่ยงตัวเลือกประเภท “ถูกทุกข้อ” หรือ “ไม่มีข้อใดถูก”
- ไม่ใช้คำถามเชิงหลอกหรืออาศัยรายละเอียดเฉพาะที่บทเรียนไม่ได้กล่าวถึง
- ตำแหน่งคำตอบที่ถูกต้องจะกระจายระหว่าง ก.–ง. และไม่เรียงเป็นรูปแบบที่เดาง่าย
- โค้ดทุกตัวอย่างต้องเป็น JavaScript ที่ถูกต้อง หรือระบุชัดเจนเมื่อคำถามต้องการถามถึงข้อผิดพลาด

- ไม่มีคำถามซ้ำกันในสาระที่วัด
